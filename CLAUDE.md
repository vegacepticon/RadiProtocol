# RadiProtocol — Agent Instructions

## Build Process

- Source: TypeScript in `src/`, compiled by esbuild → `main.js`
- CSS source: **`src/styles/`** — one file per feature (see below)
- CSS output: `styles.css` in root — **generated by esbuild, do not edit directly**
- Build: `npm run build` (production) or `npm run dev` (watch mode)
- Tests: `npm test` (vitest)

## CSS Architecture

CSS is split into per-feature files under `src/styles/`. The build concatenates them in order:

| File | Feature |
|---|---|
| `runner-view.css` | Protocol Runner UI (RunnerView) |
| `canvas-selector.css` | Canvas selector widget |
| `editor-panel.css` | Node Editor panel |
| `snippet-manager.css` | Snippet manager + placeholder chips (Phase 27) |
| `snippet-fill-modal.css` | Snippet fill-in modal |
| `loop-support.css` | Loop iteration UI |

**When adding CSS for a new feature:** create a new file in `src/styles/` and add it to the `CSS_FILES` list in `esbuild.config.mjs`. Do NOT add CSS to an unrelated feature file.

## Critical Rules for Editing Shared Files

### Never remove existing code you didn't add

When editing any file in `src/styles/`, `src/main.ts`, `src/views/editor-panel-view.ts`, or other accumulated files:

- **ONLY add or modify code relevant to the current phase**
- **NEVER delete rules, functions, or event listeners that you did not add in this phase**
- If you are unsure whether a rule/function belongs to a previous phase, leave it alone

This rule exists because recurring regressions were caused by executor agents silently deleting CSS rules and TypeScript functions while editing unrelated sections of shared files.

### CSS files: append-only per phase

When adding CSS for a new phase:
- Edit only the relevant feature file in `src/styles/`
- Add new rules at the bottom of that file with a phase comment: `/* Phase N: description */`
- Never rewrite existing sections

### After any CSS change

Always run the build to regenerate `styles.css`:
```bash
npm run build
```

Never commit `styles.css` edits directly — it is a generated file.
