**handleRunProtocolInline()** method — implement with this flow:

1. **D9 guard**: Check `this.app.workspace.getActiveFile()` — if null or not a `.md` file (check `file.extension === 'md'`), show `Notice('Open a markdown note first, then run this command.')` and return. Do NOT use `checkCallback` to hide the command (per D9 — always visible in palette).

2. **Protocol folder enumeration**: Get `this.settings.protocolFolderPath`. If empty string, show `Notice('Set a protocol folder in Settings to get started.')` and return. Resolve the folder via `this.app.vault.getAbstractFileByPath(folderPath)` — if not a `TFolder`, show `Notice('Protocol folder not found.')` and return.

3. **Enumerate .canvas files**: Recursively walk the folder's children (use the same pattern as `CanvasSelectorWidget` — `TFolder.children` filtering). Collect all `TFile` where `file.extension === 'canvas'`. If the list is empty, show `Notice("No protocol canvases found in 'Protocol' folder.")` and return (D8).

4. **Canvas picker**: Use Obsidian's `SuggestModal` pattern (not a custom modal — keep it simple). Create an inline `SuggestModal<{ file: TFile; name: string }>` that lists the canvas files by basename (without `.canvas` extension). On selection, call `openInlineRunner(selectedFile)`.

5. **openInlineRunner(file: TFile)** private helper: Import `InlineRunnerModal` from `./views/inline-runner-modal`. Instantiate: `new InlineRunnerModal(this.app, this, file.path, targetNote)` where `targetNote` is the active markdown file from step 1. Call `.open()` on it.

6. **Import**: Add `import { InlineRunnerModal } from './views/inline-runner-modal';` at the top of main.ts (after the existing view imports). Also import `SuggestModal` and `TFolder` from obsidian if not already imported.

7. Follow the existing pattern from `handleStartFromNode()` — same canvas parsing/validation approach is NOT needed here because InlineRunnerModal handles parsing/validation internally in its own `open()` method (per Plan 01). The command just picks the canvas and target note, then delegates.

8. Do NOT add `'inline'` to the `runnerViewMode` settings union. Inline is command-only per D3/D9.</action>
