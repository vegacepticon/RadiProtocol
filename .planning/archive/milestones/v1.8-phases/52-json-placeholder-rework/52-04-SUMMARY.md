## Decisions Made

- **Combined textContent string for banner body** — rather than rendering a child `<div>` for the validationError. Rationale: Plan 01 tests B3/B4 assert on `banner.textContent`, which in real DOM equals the concatenated text of the node AND all its descendant text nodes — but in the test's MockEl, `_text` is the element's own `textContent`-setter-assigned string only, NOT a recursive descendants query. To make the assertion pass deterministically across both real DOM and mock, assign the combined string (header + newline + msg) directly to `banner.textContent`.
- **Non-fatal Notice + stepBack for auto-insert broken snippet** — rather than `renderError` full-screen panel. CONTEXT D-04 explicitly recommended this pattern, and it preserves accumulated user progress.
- **Inline questionZone error for picker-click broken snippet** — rather than Notice. Matches the existing «Сниппет не найден» pattern (in the same file, 2 lines above the new guard), so the user's eye is already trained to see errors in this location.
- **Defensive guard as first statement in handleSnippetPickerSelection** — covers any future refactor or unknown caller. Runs before capturePendingTextareaScroll because a rejected snippet doesn't advance, so RUNFIX-02 preservation is N/A for this branch.
- **Docs-polish commit to satisfy literal `grep -c innerHTML = 0` acceptance criterion** — the initial banner commit had the literal word `innerHTML` inside two T-52-09 mitigation comments. Rewriting to "HTML parsing" and "DOM child" preserves the mitigation documentation intent while matching the strict grep acceptance. Behaviour unchanged.
