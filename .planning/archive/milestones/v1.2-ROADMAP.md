## Milestone Summary

**Decimal/Inserted Phases:**

- Phase 18: CSS Gap Fixes (inserted after Phase 17 to close 3 styling gaps found in milestone audit)
- Phase 19: Phase 12–14 Formal Verification (inserted to produce retroactive VERIFICATION.md artifacts)

**Key Decisions:**

- `capture-before-advance` pattern (BUG-01): syncManualEdit() called before each advance action so undo snapshot includes manual textarea edit
- `overwrite()` semantically separate from `restoreTo()`: restoreTo = undo revert of a snapshot; overwrite = inject caller's text — clearer intent at call sites
- `resolveSeparator(node)` single resolution point: node.radiprotocol_separator ?? defaultSeparator — avoids duplicated logic across 5 call sites
- `RunnerView` reconstructs `ProtocolRunner` at `openCanvas()` start — simplest way to pick up textSeparator from settings; no lazy init or observer needed
- `live textarea read` in complete-state toolbar: `previewTextarea?.value ?? capturedText` — honours final edits before copy/save/insert (replaces stale closure)
- Canvas write-back read path: `getCanvasJSON()` used in runner (live); `vault.read()` retained in EditorPanel form load (pre-existing gap, out of scope)

**Issues Resolved:**

- LAYOUT-01–04: Runner layout overhaul — auto-grow, zone ordering, button sizing, legend removal
