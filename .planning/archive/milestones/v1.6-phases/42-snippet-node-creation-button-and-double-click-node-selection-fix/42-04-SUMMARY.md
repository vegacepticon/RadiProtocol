## Decisions Made

- **`flex-wrap: wrap` over `overflow-x: auto`** — `overflow-x: auto` requires the user to discover horizontal scroll on the toolbar row. Sidebars rarely have horizontal scrollbars in Obsidian's idiom, so users unfamiliar with this pattern would miss the hidden button entirely. `flex-wrap: wrap` keeps all buttons visible without any interaction hint needed.
- **`flex-wrap: wrap` over icon-only-below-breakpoint** — An icon-only variant below a media-query breakpoint would require: (a) new CSS variants for each `.rp-create-*-btn`, (b) either a `@container` query (not supported in all Obsidian Electron versions) or a JS ResizeObserver, (c) removing the text label without breaking screen-reader accessibility (requires `aria-label` even then). For a minor UAT polish fix this is excessive.
- **Explicit `row-gap` declaration alongside `flex-wrap`** — The Phase 39 `.rp-editor-create-toolbar` rule uses `gap: var(--size-4-1)` (shorthand for both row-gap and column-gap). Modern Chromium (which Obsidian is built on) honors the shorthand across wrap boundaries, but some community themes override the `gap` shorthand with `!important`, and older Electron versions before Chromium 84 collapsed `gap` to `0` on flex wrap boundaries. The explicit `row-gap` serves as a defensive floor — it guarantees an 8px vertical gap on wrapped rows regardless of cascade interference.
- **New rule block, not merge into Phase 39 block** — CLAUDE.md's "CSS files: append-only per phase" invariant forbids editing the existing Phase 39 rule, even to add two properties. The equal-specificity source-order cascade means both blocks apply; the later block's `flex-wrap` and `row-gap` win while Phase 39's other properties remain unchanged. Zero risk of regression to existing declarations.
