## Component Inventory

### 1. Quick-Create Toolbar

**Location:** Inside `.rp-editor-panel`, rendered ABOVE the node type dropdown form. Appears in both idle and form states.

**Structure:**
```
div.rp-editor-create-toolbar
  button.rp-create-question-btn  (accent, Lucide "help-circle" icon + label)
  button.rp-create-answer-btn    (accent, Lucide "message-square" icon + label)
```

**Layout:** Horizontal flexbox row, `gap: 8px` (`--size-4-1`), `padding: 8px 0` (`--size-4-1 0`), separated from content below by `border-bottom: 1px solid var(--background-modifier-border)`.

**Button pattern:** Matches existing `radi-snippet-tree-new-btn` pattern:
- `display: inline-flex; align-items: center; gap: 4px` (`--size-2-1`)
- `padding: 6px 12px` (consistent with Obsidian `mod-cta` buttons)
- `border-radius: var(--radius-s)`
- `font-size: var(--font-ui-small)`
- `font-weight: var(--font-semibold)`
- `cursor: pointer`

**States:**

| State | Visual | Behavior |
|-------|--------|----------|
| Enabled (canvas open) | Accent background (`--interactive-accent`), white text (`--text-on-accent`) | Click creates node |
| Hover (enabled) | `filter: brightness(1.1)` | Pointer cursor |
| Disabled (no canvas) | `opacity: 0.4`, `cursor: not-allowed` | Click shows Notice: "Open a canvas first to create nodes." |
| Active (click) | `filter: brightness(0.95)` for 100ms | Visual press feedback |

### 2. Modified Idle State

**Current idle state** (from Phase 4): Shows "No node selected" with instructions.

**Phase 39 change:** The quick-create toolbar renders ABOVE the idle message. The idle text remains unchanged. The toolbar is always present when the editor panel is open, regardless of whether a node is selected.

### 3. Modified Form State

**Current form state** (from Phase 4): Shows node type dropdown + kind-specific fields + saved indicator.

**Phase 39 change:** The quick-create toolbar renders ABOVE the node type dropdown. The form content and saved indicator remain unchanged below the toolbar.

### 4. Post-Creation Auto-Load

After a successful `CanvasNodeFactory.createNode()` call:
1. The newly created node is loaded into the editor panel via `loadNode(canvasPath, newNodeId)`
2. The form renders with the new node's properties (nodeType already set by factory)
3. No additional visual indicator needed — the form itself confirms creation

## Interaction Contract

### Create Question Node

| Step | Action | Result |
|------|--------|--------|
| 1 | User clicks "Create question node" button | `CanvasNodeFactory.createNode(canvasPath, 'question', currentNodeId)` is called |
| 2a | Canvas open, API available | New question node appears on canvas adjacent to selected node (or at origin if no node selected). Editor panel loads the new node. |
| 2b | No canvas open | Obsidian Notice: "Open a canvas first to create nodes." (handled by CanvasNodeFactory) |
| 2c | API unavailable | Obsidian Notice: "Canvas API unavailable -- update Obsidian to use node creation." (handled by CanvasNodeFactory) |

### Create Answer Node

| Step | Action | Result |
|------|--------|--------|
| 1 | User clicks "Create answer node" button | `CanvasNodeFactory.createNode(canvasPath, 'answer', currentNodeId)` is called |
| 2a | Canvas open, API available | New answer node appears on canvas adjacent to selected node (or at origin if no node selected). Editor panel loads the new node. |
| 2b | No canvas open | Obsidian Notice: "Open a canvas first to create nodes." (handled by CanvasNodeFactory) |

### Anchor Node Resolution

- If a node is currently loaded in the editor (`currentNodeId` is set), use it as the anchor for positioning
- If no node is loaded (idle state), pass no anchor — factory places at canvas origin `(0, 0)`

### Canvas Path Resolution

- Obtain `canvasPath` from the active canvas leaf: `app.workspace.getLeavesOfType('canvas')` -> find active leaf -> `view.file.path`
- If no canvas leaf found, the factory shows the Notice (no UI needs to handle this separately)

## CSS Contract

**File:** `src/styles/editor-panel.css` (append new rules at the bottom)

**New CSS classes:**

```css
/* Phase 39: Quick-Create toolbar */
.rp-editor-create-toolbar {
  display: flex;
  flex-direction: row;
  align-items: center;
  gap: var(--size-4-1);           /* 8px */
  padding: var(--size-4-1) 0;    /* 8px vertical */
  margin-bottom: var(--size-4-1); /* 8px below toolbar */
  border-bottom: 1px solid var(--background-modifier-border);
  flex: 0 0 auto;
}

.rp-create-question-btn,
.rp-create-answer-btn {
  display: inline-flex;
  align-items: center;
  gap: var(--size-2-1);           /* 4px icon-to-text gap */
  padding: 6px 12px;
  border-radius: var(--radius-s);
  font-size: var(--font-ui-small);
  font-weight: var(--font-semibold);
  cursor: pointer;
  border: none;
  background: var(--interactive-accent);
  color: var(--text-on-accent);
  transition: filter 0.1s ease;
}

.rp-create-question-btn:hover,
.rp-create-answer-btn:hover {
  filter: brightness(1.1);
}

.rp-create-question-btn:active,
.rp-create-answer-btn:active {
  filter: brightness(0.95);
}

.rp-create-question-btn:disabled,
.rp-create-answer-btn:disabled {
  opacity: 0.4;
  cursor: not-allowed;
  filter: none;
}
```

**Icon sizing inside buttons:** Lucide icons rendered via `setIcon()` produce an `<svg>` element. Set `width: 16px; height: 16px` on the icon span's child SVG (Obsidian handles this by default at that container size).
