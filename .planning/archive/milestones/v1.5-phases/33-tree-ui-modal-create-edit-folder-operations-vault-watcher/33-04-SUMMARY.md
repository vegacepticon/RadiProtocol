# Phase 33 Plan 04: SnippetManagerView tree rewrite + vault watcher + modal wiring

**One-liner:** Full wholesale rewrite of `SnippetManagerView` from the legacy 735-line master-detail layout to a 542-line recursive folder-tree backed by `ItemView`, wired to `SnippetEditorModal` (Plan 03) and `ConfirmModal` (Plan 02), with a prefix-filtered 120ms-debounced vault watcher and Obsidian `Menu`-based context menus — completing the Phase 33 end-to-end snippet library UX.

## What Changed

### `src/views/snippet-manager-view.ts` — rewritten wholesale (735 → 542 lines)

- Preserved the public surface: `SNIPPET_MANAGER_VIEW_TYPE` constant, `SnippetManagerView` class name, and `ItemView` lifecycle methods. `src/main.ts` is byte-for-byte unchanged.
- Deleted the legacy master-detail renderer, id-based path logic, form panel, list panel, and all chip-editor code (now lives exclusively in `snippet-chip-editor.ts` via the modal).
- New internals:
  - `TreeNode` / `TreeNodeFolder` / `TreeNodeFile` discriminated-union model built via recursive `SnippetService.listFolder` walk with D-04 sort (folders first, then files, both A–Z via `localeCompare`).
  - `renderTree` / `renderNode(container, node, depth)` emits `.radi-snippet-tree-row` DOM with chevron, type icon (`setIcon` → folder/folder-open/file-json/file-text), label, and hover actions span.
  - Empty-library state with heading + ghost `+ Новый` button.
  - Empty-folder placeholder row («(пусто)», italic muted) rendered only when the folder is expanded (TREE-04).
- Modal wiring:
  - Row click on a file → `openEditModal(path)` → `snippetService.load` → `new SnippetEditorModal(..., { mode: 'edit', ... })` → await `modal.result` → rebuild on save.
  - Global header `+ Новый` and hover `+` buttons → `openCreateModal(folderPath)` → `new SnippetEditorModal(..., { mode: 'create', initialFolder })`.
  - Context menus via Obsidian `Menu` class: file row offers Редактировать / Удалить; folder row offers Создать сниппет здесь / Создать подпапку / Удалить папку.
  - `handleDeleteSnippet` opens `ConfirmModal` with destructive styling and the snippet name in the body.
  - `handleDeleteFolder` calls `listFolderDescendants`, builds an HTMLElement body listing up to 10 descendants + `…и ещё N элементов.` tail, opens destructive ConfirmModal, then `deleteFolder` + `Notice('Папка перемещена в корзину.')`.
  - `handleCreateSubfolder` opens ConfirmModal with label + text input body, validates non-empty / no slashes, calls `createFolder(parent + '/' + name)`, auto-expands both parent and new subfolder.
- Vault watcher:
  - Three `registerEvent(this.app.vault.on('create' | 'delete' | 'rename', ...))` subscriptions registered in `onOpen`.
  - `shouldHandle(filePath)` implements D-18 prefix filter: `filePath === root || filePath.startsWith(root + '/')`. Rename handler tests both `file.path` and `oldPath`.
  - `scheduleRedraw()` uses `window.setTimeout` with a 120ms window, coalescing bursts of vault events into a single `rebuildTreeModel` + `renderTree` call. Errors are caught and surfaced via `Notice('Не удалось обновить список сниппетов. Подробности в консоли.')`.
  - `onClose` clears the pending timer and empties `contentEl`. Vault event refs auto-detach via Obsidian's `registerEvent` contract.
- Expand/collapse state persists in `settings.snippetTreeExpandedPaths` (read + write + `saveSettings()`).

## Interpretive decisions recorded

1. **Subfolder prompt style** — The plan offered a choice between a dedicated prompt modal and a `ConfirmModal` with a custom HTMLElement body. Chose the latter: simpler, no new modal class, reuses the existing 2-button variant. The input is focused in a `setTimeout(0)` after `modal.open()` so Obsidian's own focus handling doesn't clobber it.
2. **Hover `+` button placement** — A child of `.radi-snippet-tree-actions` span with `opacity: 0 → 1` on parent-row hover. Click handler has `ev.stopPropagation()` so the row's expand-toggle click does not also fire.
3. **Row click disambiguation** — The row-level click handler checks `target.closest('button')` to ignore clicks that originated on embedded buttons, so hover-action clicks don't double-fire.
4. **onClose explicit timer clear** — Even though `registerEvent` auto-detach would prevent new fires after unmount, an already-scheduled `setTimeout` from before close would still run. Explicitly clearing it makes SYNC-03 assertable and is defensively correct.
5. **Folder delete drops expanded path** — When a folder is deleted, its path is also removed from `settings.snippetTreeExpandedPaths` so a stale cached state cannot try to re-expand a missing node after rebuild.
