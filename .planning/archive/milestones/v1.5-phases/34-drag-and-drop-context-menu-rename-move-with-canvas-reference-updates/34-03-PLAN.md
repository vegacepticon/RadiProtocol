  <behavior>
    **startInlineRename(node, labelEl):**
    - If `currentlyRenamingPath !== null` → ignore (one rename at a time)
    - Set `currentlyRenamingPath = node.path`
    - Create a transient `<input type="text">` with `radi-snippet-tree-rename-input` class
    - For file: input value = basename without extension; For folder: input value = basename
    - Replace labelEl with input, focus input, select value minus extension
    - Create a closure-scoped `let settled = false`
    - Register listeners on input:
      - keydown Enter: preventDefault, if `settled` return, set `settled = true`, call commitInlineRename(node, input.value)
      - keydown Escape: preventDefault, set `settled = true`, call cancelInlineRename(node, input)
      - blur: if `settled` return; set `settled = true`, commit (treat blur-without-Enter as commit unless value trimmed empty → cancel)
    - All listeners on the transient input use raw `addEventListener` (short-lived, manually removed in commit/cancel) — NOT registerDomEvent because the input is not part of view hierarchy lifetime.

    **commitInlineRename(node, newValue):**
    - `newValue = newValue.trim()`
    - If empty or equals old basename → cancelInlineRename (no-op)
    - try:
      - FILE: `const newPath = await snippetService.renameSnippet(oldPath, newValue);` — no canvas-sync (D-03 следствие 2)
      - FOLDER:
        - `const newPath = await snippetService.renameFolder(oldPath, newValue);`
        - `const root = settings.snippetFolderPath;`
        - `await rewriteCanvasRefs(app, new Map([[toCanvasKey(oldPath, root), toCanvasKey(newPath, root)]]));`
        - Notice with updated/skipped counts
        - Rewrite `settings.snippetTreeExpandedPaths`: entries matching `oldPath` or `oldPath + '/'` prefix → new prefix; save settings
      - FILE: Notice «Сниппет переименован.»
    - catch: Notice error in Russian; keep input visible if the error is recoverable (e.g., collision) — or revert and show Notice. Simpler: always revert to label on error, show Notice.
    - finally: `currentlyRenamingPath = null`, tree will redraw on vault 'rename' event (Phase 33 watcher); DO NOT manually call `this.renderTree()` — vault watcher handles it (120ms debounced).

    **cancelInlineRename(node, input):**
    - Replace input with the original label span (or trigger a re-render of that row)
    - `currentlyRenamingPath = null`

    **F2 keydown:**
    - Register `keydown` on each row via `this.registerDomEvent(row, 'keydown', ...)`
    - When `ev.key === 'F2'` AND row is focused (document.activeElement === row OR row contains activeElement) → preventDefault, `startInlineRename(node, labelEl)`
    - Rows need `tabindex="0"` to be focusable — add in renderNode if not already

    **Context menu «Переименовать»:**
    - File branch: add `menu.addItem(item => item.setTitle('Переименовать').setIcon('pencil-line').onClick(() => this.startInlineRename(node, labelEl)))` between «Редактировать» and «Переместить в…»
    - Folder branch: same, between «Создать подпапку» and «Переместить в…»
