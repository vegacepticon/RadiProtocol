<task type="auto">
  <name>Task 1: Create and push annotated tag 1.11.0</name>
  <files>Git repository</files>
  <read_first>
    - .planning/ROADMAP.md (Phase 74 success criteria for tag requirements)
    - .planning/phases/74-github-release-v1-11-0/74-RESEARCH.md (tag pattern from prior releases)
  </read_first>
  <action>
1. Verify the current HEAD is the `release: v1.11.0` commit from Plan 01:
   ```bash
   git log -1 --oneline
   ```
   Expected output contains `release: v1.11.0`.

2. Verify the tag does not already exist locally:
   ```bash
   git tag -l 1.11.0
   ```
   If output is non-empty, STOP and report that tag already exists.

3. Verify the tag does not already exist on the remote:
   ```bash
   git ls-remote --tags origin 1.11.0
   ```
   If output is non-empty, STOP and report that remote tag already exists.

4. Create annotated tag:
   ```bash
   git tag -a 1.11.0 -m "Release 1.11.0"
   ```

5. Verify tag was created and points to the correct commit:
   ```bash
   git show 1.11.0 --quiet
   ```

6. Push the tag to origin:
   ```bash
   git push origin 1.11.0
   ```

7. Verify remote tag exists:
   ```bash
   git ls-remote --tags origin 1.11.0
   ```
  </action>
  <verify>
    <automated>git tag -l 1.11.0 && git ls-remote --tags origin 1.11.0</automated>
  </verify>
  <acceptance_criteria>
    - `git log -1 --oneline` shows the `release: v1.11.0` commit at HEAD.
    - `git tag -l 1.11.0` returns `1.11.0` (tag exists locally).
    - `git show 1.11.0 --quiet` shows the tag message `Release 1.11.0` and points to the release commit.
    - `git ls-remote --tags origin 1.11.0` returns a non-empty line containing `refs/tags/1.11.0`.
  </acceptance_criteria>
  <done>
    Annotated tag `1.11.0` created and pushed to origin.
  </done>
</task>

<task type="auto">
  <name>Task 2: Create GitHub Release 1.11.0 with three loose assets</name>
  <files>manifest.json, main.js, styles.css</files>
  <read_first>
    - manifest.json (to confirm version before upload)
    - main.js (to confirm it exists)
    - styles.css (to confirm it exists)
    - .planning/phases/74-github-release-v1-11-0/74-RESEARCH.md (release asset pattern)
  </read_first>
  <action>
1. Verify the three asset files exist in the repo root:
   ```bash
   test -f manifest.json && test -f main.js && test -f styles.css
   ```

2. Verify `manifest.json` still says `1.11.0`:
   ```bash
   grep '"version": "1.11.0"' manifest.json
   ```

3. Create the GitHub Release using `gh` CLI:
   ```bash
   gh release create 1.11.0 `
     --title "RadiProtocol 1.11.0" `
     --notes "Release 1.11.0

   What's new:
   - Inline Runner: redundant result-export buttons removed in complete state (INLINE-CLEAN-01)
   - Runner UX: loop-exit picker button gets subtle visual accent (LOOP-EXIT-01)
   - Settings: donate section with crypto wallet addresses (DONATE-01)

   Assets: manifest.json, main.js, styles.css" `
     manifest.json main.js styles.css
   ```

   If `gh` is not available, create the release via the GitHub web UI:
   - Go to `https://github.com/vegacepticon/RadiProtocol/releases/new`
   - Choose tag: `1.11.0`
   - Release title: `RadiProtocol 1.11.0`
   - Paste the release notes above
   - Attach `manifest.json`, `main.js`, `styles.css` individually
   - Set `prerelease` to false
   - Publish release

4. Verify the release was created:
   ```bash
   gh release view 1.11.0
   ```
   Or open `https://github.com/vegacepticon/RadiProtocol/releases/tag/1.11.0` in a browser.

5. Verify the release has exactly three assets and `prerelease=false`:
   ```bash
   gh release view 1.11.0 --json assets,name,tagName,isPrerelease
   ```
   Expected JSON: `isPrerelease: false`, `assets` array length = 3, each asset name in `[manifest.json, main.js, styles.css]`.
  </action>
  <verify>
    <automated>gh release view 1.11.0 --json tagName,isPrerelease</automated>
  </verify>
  <acceptance_criteria>
    - `gh release view 1.11.0` (or web UI equivalent) shows the release exists.
    - Release title is `RadiProtocol 1.11.0`.
    - Release tag is `1.11.0`.
    - `isPrerelease` is `false`.
    - The release has exactly 3 assets: `manifest.json`, `main.js`, `styles.css`.
    - Each asset is individually downloadable (not bundled in a zip).
    - Release notes mention the three v1.11 features (Inline cleanup, loop-exit hint, donate section).
  </acceptance_criteria>
  <done>
    GitHub Release 1.11.0 published with three loose assets. BRAT can now consume the release.
  </done>
</task>
