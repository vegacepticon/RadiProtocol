**render() switch arms** (mirror RunnerView's pattern but adapt for inline):

1. **`idle`**: Show "Starting protocol…" or "Loading…" in contentEl. This is a transient state — the runner should immediately transition after `start()`.

2. **`at-node`** (question node): Render question text (`.rp-question-text`), answer buttons (`.rp-answer-btn`), snippet branch buttons (`.rp-snippet-branch-btn`), skip button (`.rp-skip-btn` if answer neighbors exist), step-back button (`.rp-step-back-btn` if `canStepBack`). Each answer button click handler:
   - Call `runner.chooseAnswer(answerId)`
   - Get the answer text from the graph node
   - Call `appendAnswerToNote(answerText)` — uses `vault.process(targetNote, content => content + separator + text)` where separator is `plugin.settings.textSeparator` (or empty if note is currently empty)
   - Call `render()` to advance to next state

3. **`awaiting-snippet-pick`**: Render a header "Choose a snippet" + embed a `SnippetTreePicker` (from `src/views/snippet-tree-picker.ts`) into the contentEl. The picker should be in `file-only` mode (since we're picking a specific snippet, not a folder). On select:
   - If `.md` snippet: load content via `plugin.snippetService.load(snippetId)`, call `runner.completeSnippet(renderedText)`, call `appendAnswerToNote(renderedText)`, call `render()`.
   - If `.json` snippet with placeholders: open snippet fill-in (see D6 handling below).
   - If `.json` without placeholders: render template, call `runner.completeSnippet(templateText)`, call `appendAnswerToNote(templateText)`, call `render()`.

4. **`awaiting-loop-pick`**: Render loop node headerText + list of body branches + exit button. Get edges from graph (same pattern as RunnerView's loop picker). Body branch buttons call `runner.chooseLoopBranch(edgeId)` then `render()`. Exit button: same flow.

5. **`awaiting-snippet-fill`**: **D6 critical** — render the snippet fill-in UI INSIDE the inline modal, NOT as a stacked Obsidian Modal. Two approaches:
   - **Approach A (preferred)**: Create a lightweight inline version of the fill-in form. Read the snippet's placeholders, render the same chip editor UI (`SnippetChipEditor` pattern) directly into `contentEl`. On submit, call `renderSnippet(snippet, filledValues)` from `snippet-model.ts`, then `runner.completeSnippet(renderedText)`, then `appendAnswerToNote(renderedText)`, then `render()`.
   - **Approach B**: Modify `SnippetFillInModal` to accept an optional `container: HTMLElement` parameter. When provided, render into that container instead of `this.contentEl` (the Modal's default). This keeps the existing Modal class unchanged for sidebar/tab mode.
   - Use Approach A — it's simpler and doesn't require modifying the existing Modal class. The fill-in form is just a series of text inputs and choice selectors — render them directly into `contentEl` with the same styling as the existing modal.

6. **`complete`**: Render final text preview (`.rp-preview-zone` with the accumulated text) + output toolbar (`.rp-output-toolbar`) with Copy, Save, and Insert buttons. Copy: copy to clipboard via `navigator.clipboard.writeText()`. Save: call `plugin.saveOutputToNote(finalText)`. Insert: call `plugin.insertIntoCurrentNote(finalText, targetNote)`.

7. **`error`**: Render error message in a styled error panel (same pattern as RunnerView's `renderError()`). No retry button — user closes modal and restarts.

**appendAnswerToNote(text: string)** implementation:
```typescript
private async appendAnswerToNote(text: string): Promise<void> {
  const currentContent = await this.plugin.app.vault.read(this.targetNote);
  const separator = currentContent.trim().length === 0 ? '' : '\n'; // or use plugin.settings.textSeparator
  const newContent = currentContent + separator + text;
  await this.plugin.app.vault.modify(this.targetNote, newContent);
}
```
Note: Use `vault.modify` (not `vault.process`) since we need to read current content first to determine separator. The WriteMutex pattern from `main.ts:insertIntoCurrentNote` should be reused if available.

**D6 snippet fill-in inline rendering**:
Create a private method `renderSnippetFillIn(snippet: JsonSnippet)` that:
1. Creates a form container in `contentEl`
2. For each placeholder: if `free-text`, render a text input; if `choice`, render checkboxes (multi-select per Phase 52)
3. Submit button: on click, collect values, call `renderSnippet(snippet, values)` from `snippet-model.ts`, then complete the snippet flow
4. Cancel button: returns to previous state (call `runner.stepBack()` if needed, then `render()`)

**Session handling**: Inline mode does NOT use session persistence (no `autoSaveSession`). The note is the buffer; if the user closes the modal mid-run, the partial output stays in the note (D3). No session file is created.

**Important**: Do NOT call `autoSaveSession()` in inline mode. The session service is sidebar/tab only. Inline mode is stateless from the plugin's persistence perspective — the note carries the state.</action>
