<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Repo → GitHub Release assets | Files committed to the default branch become the `manifest.json` downloaded by BRAT; the `author`/`authorUrl` fields are publicly visible in every installed-user's Obsidian Community Plugins UI. |
| Git tag → BRAT resolver | The string `manifest.version` must byte-match the tag name or BRAT fails to resolve assets. Mismatch creates a user-visible install error but does not leak data. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-55-01 | I (Information disclosure) | `manifest.json.author` field | mitigate | D9 locks the value to the public GitHub handle `vegacepticon` only — no real name, no email, no other PII. Acceptance criteria on Task 55-01-01 asserts `author === "vegacepticon"` verbatim; any deviation fails the automated verify. `fundingUrl` stays `""` (no donation channel → no third-party URL leak). |
| T-55-02 | I (Information disclosure) | `manifest.json.authorUrl` field | mitigate | D9 locks to `https://github.com/vegacepticon` — the existing public GitHub profile. No alternate URL permitted. Acceptance criteria asserts byte-exact match. |
| T-55-03 | S (Spoofing / collision) | git tag `1.8.0` vs. legacy `v1.8.0` | mitigate | Research confirmed only `v1.0…v1.7` exist in the repo (no `v1.8.0`). Plan 03 acceptance will additionally assert `git tag -l 'v1.8.0'` returns empty AND `git rev-parse --verify 1.8.0` succeeds — prevents silent collision if a legacy tag surfaces later. |
| T-55-04 | T (Tampering with build artifacts) | `styles.css` / `main.js` committed vs. ignored | mitigate | Handled in Plan 02 (gitignore + untrack). This plan (Plan 01) does not touch artifacts — out of scope for this threat row. |
| T-55-05 | I (Accidental version drift between three files) | manifest.json.version vs. versions.json key vs. package.json.version | mitigate | Each task has an explicit automated assertion on the exact `1.8.0` string. Plan 03 preflight script re-asserts all three together before tag creation. Drift cannot survive either gate. |

All threats MITIGATED. No `accept` or `transfer` dispositions. No credential handling, no tokens, no network input.
