# Milestone v1.1: UX & Community Release

**Status:** ✅ SHIPPED 2026-04-08
**Phases:** 8–11
**Total Plans:** 9

## Overview

v1.1 extended the plugin's UX and authoring workflow across four phases: a full-tab runner view mode, an in-runner canvas selector dropdown, an "insert into current note" output destination, and live canvas editing while the canvas is open. All four phases were UAT-approved.

## Phases

**Requirements:** RUNTAB-01, RUNTAB-02, RUNTAB-03
**Plans:** 2 plans

Plans:

**Details:**

## Milestone Summary

**Key Decisions:**

- `activateRunnerView()` D-04 deduplication pattern using `getRoot()` identity — clean leaf management across sidebar/tab mode changes
- Canvas selector rendered in `onOpen()` headerEl, not contentEl — survives render() wipes on every state transition
- `insertMutex` as separate `WriteMutex` instance on plugin — keeps file-path keying isolated from snippetService mutex
- `CanvasLiveEditor` Pattern B (getData/setData) chosen over Pattern A (direct node mutation) — cleaner API, requestSave() triggers canvas persistence
- Live-first/Strategy-A-fallback with explicit returns (T-11-06, T-11-07) — prevents dual-write and Strategy A on error
- vitest `resolve.alias` for `obsidian` — required fix for 5 Obsidian-dependent test suites; obsidian package has empty `main` field

**Issues Resolved:**
