  <how-to-verify>
    **Setup:**
    1. Reload the plugin in Obsidian (`Ctrl+R` or "Reload app without saving" from command palette).
    2. Open the Snippet Manager view from the ribbon/command palette.

    **Tree view (TREE-01..04):**
    3. Confirm the tree is rooted at `.radiprotocol/snippets/` (or whatever `snippetFolderPath` is set to).
    4. Confirm JSON snippets show a `file-json` lucide icon, Markdown snippets show `file-text`.
    5. Expand/collapse a folder by clicking; chevron swaps between `chevron-right` and `chevron-down`.
    6. Close the view and reopen it — expand state should be preserved (persistence via settings).
    7. Create an empty subfolder via context menu (see FOLDER-01 below) — the empty folder should show `(пусто)` in muted italic when expanded.

    **Global "+ Новый" button (MODAL-01, MODAL-04, MODAL-03):**
    8. Click the header "+ Новый" button. Modal title should be `Новый сниппет`.
    9. «Папка» dropdown should pre-select the snippet root path.
    10. Type toggle should show `JSON` and `Markdown` segments. Default is JSON; chip editor should be visible below the name field.
    11. Click `Markdown`. Content area should swap to a plain `<textarea>` with placeholder `Введите текст сниппета…`.

    **Folder hover + Новый (FOLDER-03):**
    12. Hover over a folder row — a small `+` button should fade in on the right side of the row.
    13. Click it. The modal should open with «Папка» pre-selected to that folder.

    **Create a snippet:**
    14. In create mode, give the snippet a unique name, type «Создать». The modal closes, a `Notice` may appear, and the new file should appear in the tree under the chosen folder.
    15. Try creating with a name that already exists (same folder) — the Save button should be disabled and an inline error `Файл с таким именем уже существует в этой папке.` should appear under the name field.

    **Edit an existing snippet (TREE-03, MODAL-02, MODAL-06/07):**
    16. Click an existing JSON snippet row. Modal opens in edit mode, title `Редактирование: {name}`. Chip editor should show the existing placeholders.
    17. Click an existing MD snippet row. Modal opens with the textarea pre-filled from the file.
    18. In edit mode, the JSON/MD toggle should be hidden or locked (type cannot change).

    **Move via «Папка» (D-09 — the single D-09 wiring path this phase ships):**
    19. Open a JSON snippet that you know is referenced by a SnippetNode in a `.canvas` file. Change its «Папка» dropdown to a different folder. Click `Сохранить`.
    20. A `Notice` should appear: `Сниппет перемещён. Обновлено канвасов: X, пропущено: Y.`
    21. Open the referring canvas in Obsidian Canvas view, and then run the protocol from that canvas — the snippet should still resolve correctly (no broken link). If X was 0 and the canvas is still broken, report the issue — D-09 wiring failed.

    **Unsaved-changes guard (MODAL-08):**
    22. Open any snippet for edit, type a character into the name field, then press Esc.
    23. A ConfirmModal should appear with title `Несохранённые изменения`. Verify three buttons present, ordered left→right: `Не сохранять` · `Отмена` · `Сохранить`.
    24. Click `Отмена` — you return to the editor modal.
    25. Press Esc again and click `Не сохранять` — the editor closes without saving.

    **Context menus:**
    26. Right-click a snippet row — menu should show `Редактировать` / (separator) / `Удалить`.
    27. Right-click a folder row — menu should show `Создать сниппет здесь` / `Создать подпапку` / (separator) / `Удалить папку`.

    **Folder create (FOLDER-01):**
    28. From a folder's context menu, choose `Создать подпапку`. Enter a name in the prompt, confirm.
    29. New subfolder appears in the tree expanded immediately (it should be in the expanded-paths list).

    **Folder delete with confirm list (FOLDER-02, D-15):**
    30. Pick a folder with 12+ descendants. Right-click → `Удалить папку`.
    31. The ConfirmModal body should list up to 10 items, then show `…и ещё 2 элементов.` tail.
    32. Click `Удалить папку`. Notice: `Папка перемещена в корзину.` Folder disappears from tree.
    33. Confirm the folder is now in Obsidian's `.trash/` (or however your platform handles it).

    **Snippet delete (DEL-02, DEL-03):**
    34. Right-click a snippet → `Удалить`. Confirm dialog shows the snippet name. Click `Удалить`.
    35. Notice: `Сниппет перемещён в корзину.` Row disappears.
    36. Open the Protocol Runner → snippet picker in the same folder. Confirm the deleted snippet is no longer listed (DEL-03 — tree and picker in sync).

    **External vault sync (SYNC-01..03):**
    37. Open the Obsidian file explorer (sidebar). Navigate to `.radiprotocol/snippets/`. Right-click a snippet → Rename to something else.
    38. Within ~150ms the tree view should redraw showing the new name and hiding the old.
    39. Create a new file inside `.radiprotocol/snippets/something/` from the file explorer — tree redraws.
    40. Create a file OUTSIDE `.radiprotocol/snippets/` (e.g. in vault root) — tree should NOT redraw (prefix filter works).
    41. Close the Snippet Manager view, reopen it. Everything still works.
    42. Close Obsidian entirely, reopen. Expand state persisted; tree still renders correctly.

    **Visual polish:**
    43. Switch Obsidian theme (light/dark or a community theme). Tree and modal should adapt because they use only `var(--...)` tokens — no hardcoded colors.
    44. Sanity: no English strings anywhere in the tree view or modals.
