    - Fields rendered in order: Type toggle (create only, segmented JSON/Markdown) · «Папка» dropdown · Name text input · Content section (chip editor OR textarea based on kind).
    - «Папка» dropdown: populated from walking the snippet root via `plugin.snippetService.listFolderDescendants(snippetFolderPath)` + the root itself; options sorted A-Z. Pre-selected to `options.initialFolder` (create) or `dirname(snippet.path)` (edit).
    - Name input: pre-filled from `options.snippet?.name ?? ''`; on every input, re-derive `candidatePath = folder + '/' + slugifyLabel(name) + '.' + (kind === 'json' ? 'json' : 'md')`, then pre-flight `plugin.snippetService.exists(candidatePath)` **skipping the current snippet's own path when in edit mode**. If collision, show inline error «Файл с таким именем уже существует в этой папке.» beneath the name input (use `--text-error`) and disable Save. Debounce the exists check by 150ms so fast typists don't thrash I/O.
    - JSON mode: `mountChipEditor(contentContainer, draft as JsonSnippet, () => { hasUnsavedChanges = true; })`. Store the ChipEditorHandle for destroy() on close.
    - MD mode: a plain textarea bound to `(draft as MdSnippet).content`. `input` listener sets `hasUnsavedChanges = true`.
    - Type toggle (create only): switching kind swaps the content region between chip editor and textarea, resetting the draft to an empty JsonSnippet or MdSnippet. Edit mode: toggle is NOT rendered; instead show a static type label from the on-disk kind.
    - Save button: `Создать` (create) or `Сохранить` (edit), class `mod-cta`. Disabled while collision error is shown. Clicking calls handleSave().
    - Cancel button: `Отмена`, class neutral. Clicking calls handleCancel().
    - Close via Esc / click-outside is intercepted: if `hasUnsavedChanges`, open a `ConfirmModal` guard (3-button variant: `discardLabel: 'Не сохранять'`, `cancelLabel: 'Отмена'`, `confirmLabel: 'Сохранить'`, title: 'Несохранённые изменения', body: `Изменения в сниппете «{name}» не сохранены. Что сделать?`). Await result:
      - 'confirm' → run handleSave(); if save succeeds, close with saved result
      - 'discard' → resolve result with `{ saved: false }` and close
      - 'cancel' → abort the close (stay in the modal)
    - handleSave pipeline (D-09):
      1. Build `draft` from current form state. Compute `newPath = folder + '/' + slugifyLabel(name) + '.' + ext`.
      2. If mode === 'create' OR mode === 'edit' with `newPath === snippet.path` (no move): call `plugin.snippetService.save(draft)`. Resolve with `{ saved: true, snippet: draft, movedFrom: null }`. Close.
      3. If mode === 'edit' AND newPath !== snippet.path (move-on-save): call `save(draft)`, then `delete(oldPath)`, then `rewriteCanvasRefs(app, new Map([[oldPath, newPath]]))`. Show `new Notice('Сниппет перемещён. Обновлено канвасов: ' + result.updated.length + ', пропущено: ' + result.skipped.length + '.')`. Resolve with `{ saved: true, snippet: draft, movedFrom: oldPath }`. Close.
      4. On any thrown error: show inline error `Не удалось сохранить: {errorMessage}. Попробуйте ещё раз.` under the button row; do NOT close; do NOT resolve.
    - handleCancel: if hasUnsavedChanges, open unsaved-changes guard ConfirmModal (same as close-intercept path). Otherwise resolve `{ saved: false }` and close immediately.
    - onClose: if result still unresolved (e.g. Obsidian killed the modal), safeResolve `{ saved: false }` and destroy chip editor handle if present.
