# Plan 75-06 — Test consolidation with zero silent behavior loss

## Objective

Consolidate duplicated runner/inline render assertions into shared renderer fixtures while preserving host-only tests.

## Scope

Create/modify:

- `src/__tests__/runner/runner-renderer.shared.test.ts`
- `src/__tests__/runner/runner-renderer-host-fixtures.ts`
- `src/__tests__/views/inline-runner-modal.test.ts`
- `src/__tests__/views/inline-runner-modal-loop-body-file-bound.test.ts`
- `src/__tests__/views/inline-runner-modal-output-toolbar.test.ts`
- possibly `src/__tests__/RunnerView.test.ts` or runner host tests if useful

Do not touch these unless strictly necessary, because they are host concerns:

- `src/__tests__/inline-runner-layout.test.ts`
- `src/__tests__/views/inline-runner-position.test.ts`

## Zero-loss method

Before deleting any assertion:

1. Map old assertion to a new shared renderer assertion, or
2. Keep it in a host-specific test, or
3. Document it as obsolete in `75-VERIFICATION.md` with reason.

## LOC target

DEDUP-02 baseline inline test LOC is `2222`. Post-consolidation target is `<=1555 LOC` across the five DEDUP-02 files plus the new shared replacement test file.
