# Phase 78-02: GitHub Actions CI Workflow

## Goal
Create a GitHub Actions workflow that runs on every push to `main` and every PR, executing the full build + lint + test pipeline.

## Steps

### Step 1: Create `.github/workflows/ci.yml`
Standard Node.js CI workflow:
- Name: `CI`
- Triggers: `push` to `main` branches, `pull_request`
- Node version: 18 (per ROADMAP project baseline)
- Steps:
  1. Checkout code
  2. Setup Node.js 18
  3. `npm ci`
  4. `npm run build`
  5. `npm run lint`
  6. `npm test`

### Step 2: Validate YAML syntax
```bash
python3 -c "import yaml; yaml.safe_load(open('.github/workflows/ci.yml'))"
```
Or use `actionlint` if available.

### Step 3: Verify workflow structure
Check that:
- File parses as valid YAML
- Contains `on.push.branches` with `main`
- Contains `on.pull_request`
- Contains `npm ci`, `npm run build`, `npm run lint`, `npm test` in steps

## Constraints
- No custom actions needed — standard `actions/checkout` and `actions/setup-node`
- No caching required for initial version (can be added later)
- No matrix strategy — single Node 18 run
