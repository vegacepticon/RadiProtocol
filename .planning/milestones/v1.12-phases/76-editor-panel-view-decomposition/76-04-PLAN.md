# Plan 76-04 — Extract snippet-form

## Goal
Move Snippet node form rendering to `snippet-form.ts` without breaking `SnippetTreePicker` lifecycle, file/folder selection semantics, or Phase 63 snippet-label sync.

## Scope

Create:
- `src/views/editor-panel/forms/snippet-form.ts`

Modify:
- `src/views/editor-panel-view.ts`
- `_shared.ts` if `setSnippetTreePicker` or picker-related context typing needs adjustment

## Implementation steps

1. Keep this in `EditorPanelView.buildKindForm` before delegation:
   - if `this.snippetTreePicker !== null`, unmount it and set null
2. Add `renderSnippetForm(container, nodeRecord, ctx)`.
3. Inside `snippet-form.ts`, preserve current picker construction:
   - root path from plugin settings
   - initial selection from node data
   - folder-only/file selection semantics exactly as existing arm
   - pending edits for `subfolderPath`, `snippetPath`, `text`, and label/separator fields exactly as existing arm
4. After constructing picker, call `ctx.setSnippetTreePicker(picker)`.
5. Preserve existing fire-and-forget mount semantics: `void picker.mount()`.
6. Preserve `registerFieldRef('radiprotocol_snippetLabel', textarea)`.
7. Replace only the `snippet` switch arm with delegation.

## Critical invariants

- Old picker unmount happens before delegation.
- New picker instance is registered back to the dispatcher.
- Folder selection clears file-bound snippet path exactly as before.
- File selection clears folder-bound subfolder path exactly as before.
- `pendingEdits.text` mirror behavior remains unchanged.
- No async form-rendering dependency is introduced.
