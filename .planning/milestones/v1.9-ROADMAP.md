## Milestone Summary

### Key Decisions

- **Accumulator-diff pattern reused in Inline mode (Phase 59)** — instead of duplicating separator-aware append logic, `appendDeltaFromAccumulator(beforeText)` mirrors the sidebar `handleAnswerClick` helper's before/after snapshot approach. Rationale: single pattern, zero drift between modes. Outcome: ✓ Good.
- **Phase 54 D6 reversal in Phase 59 (INLINE-FIX-05)** — reverted the v1.8 decision to render fill-in UI inline inside the floating modal; JSON snippets now open a real `SnippetFillInModal` above the floating runner with `isFillModalOpen` D1 gate blocking `handleActiveLeafChange`. Rationale: parity with sidebar + tab modes was the stronger UX contract. Outcome: ✓ Good.
- **Phase 60-01/60-02 merged in single commit (`e73d829`)** — position state contract + drag wiring shipped together to avoid a half-wired modal state. Rationale: intermediate commit would leave drag events firing against non-persistent state. Outcome: ✓ Good.
- **Compact layout by default, no resize handle (Phase 60)** — chose tighter padding / reduced preview height over adding a user-adjustable resize handle. Rationale: overlap problem solved by compact default; resize is additive polish deferred. Outcome: ✓ Good — UAT approved.
- **Viewport-bounds clamp on restore (Phase 60)** — saved coordinates that would place modal off-screen are clamped at restore time, not at save time. Rationale: monitor/resolution changes happen between sessions; clamp-at-restore handles the geometry that actually exists now. Outcome: ✓ Good.
- **Reusable `FolderSuggest` on `AbstractInputSuggest` (Phase 61)** — single class attached to 3 fields, not per-field custom suggesters. Rationale: matches Templater reference pattern; zero duplication. Outcome: ✓ Good.
- **Session folder excluded from autocomplete (Phase 61)** — SETTINGS-01 scope limited to Protocols/Snippets/Output fields. Rationale: explicit scope boundary; Session path rarely changed. Outcome: ✓ Good — no user friction reported.
- **Unprefixed tag `1.9.0` (Phase 62)** — BRAT convention continues from v1.8.0 (previous milestones used `v1.0`..`v1.7` with v-prefix). Rationale: BRAT manifest/versions.json convention; prior milestone set the precedent. Outcome: ✓ Good.
- **D10 Phase 60 UAT gate as first runbook section (Phase 62)** — release runbook structurally diverges from Phase 55 by placing the Phase 60 UAT gate before any release-asset step. Rationale: A3 guardrail — release notes cannot include bullets for phases that hadn't passed UAT yet. Outcome: ✓ Good — runbook executed cleanly.
- **Single 1.9.0 release at end of milestone (no patch releases)** — mirrors v1.8 cadence; no interim 1.9.1/1.9.2 betas. Outcome: ✓ Good.
