#### Plan 01: Chip Renderer + DnD + Auto-Save

- Added `PH_COLOR` constant mapping placeholder type to CSS custom property colour
- Replaced `renderPlaceholderRow()` with `renderPlaceholderChip()` — chip with coloured left bar, human-readable label, type badge, `⠿` drag handle, `×` remove button
- HTML5 native DnD: 6-event handler set per chip (`dragstart`/`dragover`/`dragenter`/`dragleave`/`drop`/`dragend`), correct `splice(from,1)` then `splice(to,0,moved)` pattern
- `autoSaveAfterDrop()` calls `snippetService.save(draft)`, syncs `this.snippets`, shows `Notice('Snippet saved.')`
- UUID guard in `autoSaveAfterDrop()` — skips save for unsaved drafts
- 5 code review issues (WR-01–05) found and fixed before UAT

## Key Decisions

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| HTML5 native DnD (no library) | Chips are recreated on each re-render so `addEventListener` is correct; no drag library overhead | ✓ Good |
| `border-left` inline style for colour bar | Avoids per-type CSS class proliferation; `PH_COLOR` maps type to CSS var at render time | ✓ Good |
| UUID guard before auto-save | Prevents accidental save of snippets with unsaved placeholder IDs | ✓ Good — WR-03 fix |
| `autoSaveAfterDrop()` bare `addEventListener` | Chip elements are ephemeral (recreated on rerender); `registerDomEvent` is for persistent elements only | ✓ Good — WR-05 fix |
