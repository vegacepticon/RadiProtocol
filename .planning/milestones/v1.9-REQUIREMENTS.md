## v1.9 Requirements

### Inline Runner Fixes

- [x] **INLINE-FIX-01**: Inline Runner correctly resolves multi-segment Protocols folder paths (e.g. `templates/ALGO`) — canvas files inside a nested folder are found and listed identically to canvas files in a root-level folder
  - **Outcome:** Validated — `resolveProtocolCanvasFiles(vault, folderPath)` helper handles trailing slashes, backslashes, and falls back to `vault.getFiles()` prefix scan when `getAbstractFileByPath` returns null. 7 tests GREEN.
- [x] **INLINE-FIX-02**: Inline Runner persists drag-position in workspace state — after tab switch (or plugin reload), modal reappears at the last user-set coordinates, clamped to current viewport bounds (never clipped off-screen)
  - **Outcome:** Validated — persistent position state + finite-coordinate guards + viewport-bounds clamp-on-restore. UAT approved on real Obsidian vault.
- [x] **INLINE-FIX-03**: Inline Runner uses a compact default layout — base modal footprint reduced (tighter padding / reduced preview height) so it does not visually overlap the active note text area at default Obsidian window size
  - **Outcome:** Validated — compact draggable CSS overrides in `src/styles/inline-runner.css`. UAT approved.
- [x] **INLINE-FIX-04**: Inline Runner applies the configured separator when inserting snippet content (matches sidebar-runner behavior — no glued concatenation between prior text and inserted snippet)
  - **Outcome:** Validated — `appendDeltaFromAccumulator(beforeText)` helper wired into MD and JSON-zero-placeholder arms. Per-node separator override honoured. D7 parity check (inline output matches sidebar output) green.
- [x] **INLINE-FIX-05**: Inline Runner opens SnippetFillInModal for JSON snippets with placeholders — fill-in fields + live preview + tab navigation work identically to sidebar-runner behavior
  - **Outcome:** Validated — `SnippetFillInModal` imported into Inline mode with `isFillModalOpen` D1 gate + defensive `fillModal.close()` disposal in `close()`. Phase 54 D6 reversal enacted. All 13 focused tests GREEN.

### Settings UX

- [x] **SETTINGS-01**: All path input fields in plugin settings (Protocols folder, Snippets folder, Output folder) render a folder autocomplete dropdown of existing vault folders as the user types — Templater-style via Obsidian `AbstractInputSuggest` pattern
  - **Outcome:** Validated — single reusable `FolderSuggest` class attached to 3 SETTINGS-01 fields (Session folder deliberately excluded as scope boundary). 13 focused tests + full 707-passed suite + live Obsidian UAT approved.
