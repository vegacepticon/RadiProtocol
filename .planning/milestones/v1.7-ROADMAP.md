# Milestone v1.7: Loop Rework & Regression Cleanup

**Status:** ✅ SHIPPED 2026-04-18
**Phases:** 43-46
**Total Plans:** 18

## Overview

Collapsed the legacy `loop-start` / `loop-end` node pair into a single unified `loop` node spanning graph model, parser, validator, runtime, Node Editor, NodePickerModal, color map, and session persistence. Introduced break-compatibility migration path — legacy canvases surface a plain-language validator error in the RunnerView error panel prompting authors to rebuild loops manually. Retired the per-loop iteration cap (`maxIterations`) as redundant. Excised the regressed `free-text-input` node kind from every layer of the plugin — model, parser, validator, runner, editor, picker, color map, and test fixtures — restoring the original v1.0 decision after the v1.2 regression.

## Milestone Summary

**Decimal Phases:** None. All phases numbered sequentially 43-46.

**Key Decisions:**

- **Break-compatibility over auto-migration** — legacy `loop-start`/`loop-end` canvases produce a validator error; authors rebuild loops manually with clear guidance. Rationale: auto-migration semantics (which edge becomes «выход», how to collapse body) were judged ambiguous enough that explicit author action is safer than silent rewriting.
- **Edge label «выход» as exit discriminator** — loop exit branch identified by the Russian edge label (5 chars, case-sensitive). Rationale: domain-correct; matches the protocol-authoring language and avoids a second special edge property.
- **One-step picker** — all body-branch labels plus «выход» rendered together above an author-editable `headerText`. Rationale: simpler than two-step (pick iterate-or-exit → pick branch); every iteration re-presents the same picker.
- **`maxIterations` retired entirely** — no per-loop or global cap in v1.7; `ProtocolRunner.maxIterations` RUN-09 cycle guard preserved. Rationale: author responsibility, not engine policy.
- **Nested loops via existing `LoopContext` stack** — no architectural rework needed; B1 re-entry guard handles inner-«выход» landing on outer loop without pushing a second frame.
- **TypeScript exhaustiveness as excision forcing function** — Phase 46 used `Record<RPNodeKind, string>` + switch exhaustiveness to mechanically find every residual `free-text-input` reference, rather than manual grep sweeps.
- **Fixture retained, role flipped** — `free-text.canvas` kept byte-identical but its semantic role flipped from RUN-04 happy-path to CLEAN-02 rejection proof. Rationale: ROADMAP SC#4 "removed or rewritten" satisfied via rewrite path; avoids churn in git history.
