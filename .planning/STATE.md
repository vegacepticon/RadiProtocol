### v1.7 Design Decisions (locked during /gsd-explore)

1. Break-compatibility chosen over auto-migration вАФ old `loop-start`/`loop-end` canvases produce a validator error, author rebuilds loops
2. Exit edge identified by edge label ¬Ђ–≤—Л—Е–Њ–і¬ї
3. Multiple body branches allowed; each iteration re-presents the picker
4. One-step picker combining body-branch labels + ¬Ђ–≤—Л—Е–Њ–і¬ї
5. `maxIterations` removed entirely; no per-loop or global cap
6. Nested loops keep working via the existing `LoopContext` stack
7. Loop node owns an editable `headerText` rendered above the picker

### Standing Pitfalls

1. Never modify `.canvas` while open in Canvas view (Strategy A) unless Pattern B live-editor is used
2. `vault.modify()` race conditions вАФ use `WriteMutex` (async-mutex) per file path
3. No `innerHTML` вАФ use DOM API and Obsidian helpers
4. No `require('fs')` вАФ use `app.vault.*` exclusively
5. `loadData()` returns null on first install вАФ always merge with defaults
6. `console.log` forbidden in production вАФ use `console.debug()` during dev
7. CSS files are append-only per phase вАФ edit only the relevant feature file in `src/styles/`
8. Shared files (main.ts, editor-panel-view.ts, snippet-manager-view.ts) вАФ only modify code relevant to the current phase
9. Real-DOM vs mock-DOM parent lookup: always use `parentElement` first, `.parent` mock fallback second
10. v1.7-specific: LOOP rework must delete the old iteration cap (RUN-07) вАФ do not carry the `maxIterations` field forward for any reason
