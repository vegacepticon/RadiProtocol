<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| EditorPanelView.saveNodeEdits() → CanvasLiveEditor.saveLive() | Caller-supplied edits payload flows into live canvas mutation |
| CanvasLiveEditor.saveLive() → vault.modify() fallback | If live path returns false, execution falls through to the Strategy A write path |
| main.ts onunload() → CanvasLiveEditor.destroy() | Plugin lifecycle controls timer cleanup |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-06 | Tampering | saveNodeEdits() dual-write path: if saveLive() returns true, vault.modify() must NOT also be called | mitigate | `if (savedLive) { ... return; }` early return before vault.modify() block; explicit return prevents fall-through |
| T-11-07 | Denial of Service | requestSave() error path: if catch block shows Notice but doesn't return, Strategy A also runs on the open canvas | mitigate | catch block includes explicit `return` after Notice — Strategy A fallback only runs on `saveLive() returns false`, not on thrown error |
| T-11-08 | Tampering | canvasLiveEditor.destroy() not called on unload — timers fire on dead plugin | mitigate | destroy() added to onunload() before console.debug; verified by grep in acceptance criteria |
</threat_model>
