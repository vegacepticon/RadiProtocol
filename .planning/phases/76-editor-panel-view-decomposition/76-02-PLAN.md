# Plan 76-02 — Extract question-form + text-block-form

## Goal
Move the two growable-textarea-only form bodies to per-kind modules while preserving Phase 63 field registration.

## Scope

Create:
- `src/views/editor-panel/forms/question-form.ts`
- `src/views/editor-panel/forms/text-block-form.ts`

Modify:
- `src/views/editor-panel-view.ts`
- `src/views/editor-panel/forms/_shared.ts` if the textarea option type needs widening

## Implementation steps

1. Add `renderQuestionForm(container, nodeRecord, ctx)`:
   - preserve label/description/copy exactly
   - preserve dual pending write:
     - `radiprotocol_questionText`
     - `text`
   - preserve `registerFieldRef('radiprotocol_questionText', textarea)`
   - preserve autosave scheduling via `ctx.scheduleAutoSave()`
2. Add `renderTextBlockForm(container, nodeRecord, ctx)`:
   - preserve content textarea behavior
   - preserve `registerFieldRef('radiprotocol_content', textarea)` if present in source arm
   - preserve separator dropdown behavior
3. Replace `question` and `text-block` switch-arm bodies with delegations.
4. Keep `renderGrowableTextarea` in `EditorPanelView`, exposed through `ctx.renderGrowableTextarea`.

## Critical invariants

- Field keys must remain byte-identical.
- Form modules must not call `saveNodeEdits` directly.
- Form modules must not import `EditorPanelView`.
