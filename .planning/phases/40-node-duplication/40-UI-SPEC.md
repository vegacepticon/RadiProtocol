## Component Inventory

### 1. Duplicate Node Button

**Location:** Inside the existing `.rp-editor-create-toolbar` toolbar (Phase 39), appended after the "Create answer node" button.

**Visual spec:**
- CSS class: `.rp-duplicate-btn`
- Icon: `copy` (Lucide) via `setIcon()`, placed before label text
- Label text: `Duplicate node`
- Style: Identical to `.rp-create-question-btn` / `.rp-create-answer-btn` — same padding (6px 12px), border-radius (`var(--radius-s)`), font-size (`var(--font-ui-small)`), font-weight (`var(--font-semibold)`), background (`var(--interactive-accent)`), color (`var(--text-on-accent)`)
- Hover: `filter: brightness(1.1)`
- Active: `filter: brightness(0.95)`
- Disabled: `opacity: 0.4; cursor: not-allowed; filter: none`

**Enabled/disabled logic:**
- Enabled when a node is currently loaded in the editor panel (i.e., `this.currentNodeId` is set AND canvas is open)
- Disabled when no node is selected or no canvas is open

**Interaction flow:**
1. User clicks "Duplicate node" button
2. System reads all properties from the currently loaded node via `canvasNode.getData()`
3. System calls `CanvasNodeFactory.createNode()` with the source node's `radiprotocol_nodeType` and source node ID as anchor
4. System copies all `radiprotocol_*` properties from source to new node via `setData()`
5. System does NOT copy edges (DUP-02)
6. New node appears on canvas offset to the right of the source (DEFAULT_NODE_WIDTH + NODE_GAP = 290px right)
7. Editor panel loads the new node for immediate editing
8. Obsidian Notice: not shown on success (silent success — node visually appears)

### 2. States

| State | What user sees |
|-------|---------------|
| No node selected | Duplicate button is disabled (grayed out, 0.4 opacity) |
| Node selected, canvas open | Duplicate button is enabled (accent color) |
| No canvas open | Duplicate button is disabled |
| Duplication succeeds | New node appears on canvas, editor panel switches to new node |
| Canvas API unavailable | Obsidian Notice: "Canvas API unavailable -- update Obsidian to use node creation." (reuses Phase 38 notice) |
| No canvas open on click | Obsidian Notice: "Open a canvas first to create nodes." (reuses Phase 38 notice) |

## Layout Contract

The duplicate button is appended to the existing `.rp-editor-create-toolbar` flex row. No new containers are created.

```
.rp-editor-create-toolbar (existing Phase 39)
  [icon] Create question node    (existing)
  [icon] Create answer node      (existing)
  [icon] Duplicate node          (NEW - Phase 40)
```

The toolbar uses `display: flex; flex-direction: row; gap: var(--size-4-1)` (8px). Three buttons will fit in the standard Obsidian sidebar width (~300px). If the sidebar is narrower, buttons wrap naturally due to flex-wrap (no explicit wrap needed -- buttons are compact enough at ~140px + ~130px + ~120px).

## Interaction Sequence

```
1. User selects a node on canvas (any node type)
2. Editor panel loads node properties
3. Duplicate button becomes enabled
4. User clicks "Duplicate node"
5. onDuplicate() reads source node getData()
6. createNode() creates new node with source nodeType, anchored to source
7. All radiprotocol_* properties copied to new node via setData()
8. Edges NOT copied (DUP-02)
9. canvas.requestSave() persists changes
10. Editor panel loads new node (same pattern as onQuickCreate)
```
