# Phase 4 — UI Design Contract

> Visual and interaction contract for the Canvas Node Editor Side Panel.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Obsidian plugin — plain DOM + Obsidian API) |
| Preset | not applicable |
| Component library | Obsidian `Setting` API (`addText`, `addDropdown`, `addTextArea`, `setHeading`) |
| Icon library | Obsidian built-in icon set via `getIcon()` — icon: `pencil` for this view |
| Font | Obsidian system font stack via `var(--font-interface)` for UI; `var(--font-monospace)` for content previews |

Design token source: Obsidian CSS custom properties. No external token file. All values resolved at runtime by the active Obsidian theme.

## Spacing Scale

All spacing uses Obsidian CSS custom properties mapped to an 8-point scale. From Phase 3 (carried forward, source: `src/styles.css`):

| Token | Obsidian Variable | Value | Usage |
|-------|------------------|-------|-------|
| xs | `var(--size-2-1)` | 4px | Icon gaps, inline element gaps |
| sm | `var(--size-4-1)` | 8px | Compact element margins, divider margins |
| md | `var(--size-4-2)` | 16px | Section padding, form field padding |
| lg | `var(--size-4-3)` | 24px | Section breaks between form groups |
| xl | `var(--size-4-4)` | 32px | Major layout gaps (not used in sidebar panels) |

Exceptions:
- Save button toolbar: full-width, no side margin — matches `.rp-output-toolbar` pattern from Phase 3
- Form heading separators (`Setting.setHeading()`): spacing is controlled by Obsidian's `Setting` component internals; do not add extra margin
- Touch/click target minimum: 40px height on all interactive controls (carried from Phase 3 `.rp-answer-btn`)

## Color

This is an Obsidian plugin. There are no hardcoded hex values. All color is resolved via Obsidian CSS custom properties, inheriting the active user theme (dark or light). The 60/30/10 split is expressed through Obsidian's semantic color variables.

| Role | Obsidian Variable | Split | Usage |
|------|------------------|-------|-------|
| Dominant surface | `var(--background-primary)` | 60% | View background, textarea backgrounds |
| Secondary surface | `var(--background-secondary)` | 30% | Form section backgrounds (applied via `Setting` component), idle state container |
| Accent | `var(--interactive-accent)` | 10% | Save button (primary CTA only) |
| Destructive | `var(--text-error)` / `var(--background-modifier-error)` | accent on error elements only | Canvas-open blocking notice, field validation errors |
| Muted text | `var(--text-muted)` | — | Idle state body copy, form descriptions, placeholder text |
| Border | `var(--background-modifier-border)` | — | Section dividers, field borders |

Accent reserved for: the "Save changes" button only. No other element in the editor panel uses accent color.

Destructive color reserved for: the canvas-open blocking notice ("Close the canvas before editing") and inline field validation error messages. Not used for the notice that appears after a successful save.

## Component Inventory

All components are Obsidian-native. No third-party components.

| Component | Obsidian API | Phase 3 Precedent | Usage |
|-----------|-------------|-------------------|-------|
| Panel container | `ItemView.contentEl` with `.rp-editor-panel` class | `.rp-runner-view` | Outer scroll container for the full form |
| Form heading | `new Setting(container).setHeading().setName(text)` | Settings tab | Node kind section separator |
| Text input | `new Setting(container).addText(cb)` | Settings tab | Single-line string fields |

| Dropdown | `new Setting(container).addDropdown(cb)` | Settings tab | Node kind selector (`RPNodeKind`) |
| Number input | `new Setting(container).addText(cb)` with `type="number"` hint | Settings tab | `radiprotocol_maxIterations` field |

| Icon | `setIcon(el, 'pencil')` | Phase 3 ribbon | Panel tab icon |

Layout pattern: single-column vertical scroll. The `Setting` API renders each field as a full-width labeled row. No multi-column layout.

## Interaction Contract

### State Machine: EditorPanelView States

| State | Trigger | Rendered Content |
|-------|---------|-----------------|
| `idle` | `onOpen()` before any node loaded | Idle message: "Select a canvas node and choose 'Edit RadiProtocol properties' from the context menu." |
| `loading` | `loadNode()` called | Panel empties, then immediately renders form (no async spinner needed — `vault.read()` is fast for small canvas files) |
| `form` | Node data loaded successfully | Form with node type selector + kind-specific fields + Save button |
| `blocked` | User clicks Save while target canvas is open | Notice: "Close the canvas before editing node properties." Panel remains in `form` state — no data lost |
| `save-success` | `vault.modify()` completes without error | Notice: "Node properties saved." Panel remains in `form` state with current values |
| `save-error` | `vault.modify()` throws, or node ID not found | Notice: "Could not save — {reason}." Panel remains in `form` state with current values |

### Form Behavior

- Node type selector appears first, above all kind-specific fields.
- Changing the node type selector immediately re-renders the kind-specific field section below it without clearing the selector itself.
- All fields collect edits into a `pendingEdits` object in memory. No field auto-saves on change.
- Save button triggers the write-back sequence (canvas-open check → `vault.modify()`).
- Clicking Save does NOT close or reset the panel. The user stays in the `form` state to verify or make further edits.
- "Cancel" button is not provided. The user can reload the node by re-triggering the context menu.

### Context Menu Integration (EDIT-05)

- Right-clicking any canvas node appends a separator and a "Edit RadiProtocol properties" item to the node's context menu.
- Clicking the menu item: (1) opens or reveals `EditorPanelView` in the right sidebar, (2) calls `view.loadNode(filePath, nodeId)`.
- If the panel is already open, it is reused (no second instance). The `detachLeavesOfType` + `getRightLeaf` pattern from Phase 3 is carried forward.
- The context menu item appears on ALL canvas nodes, not just those already marked as RadiProtocol nodes. The form will show the node type as unset and allow assignment.

### Write-Back Strategy (EDIT-04) — Strategy A

The decided strategy is "require canvas closed" (A4/A5 from STATE.md confirmed in RESEARCH.md).

- Before writing: call `isCanvasOpen(filePath)` using `getLeavesOfType('canvas').some(...)`.
- If open: `new Notice('Close the canvas before editing node properties.')` — abort write, panel stays in `form` state.
- If closed: `vault.read()` → `JSON.parse()` → patch `radiprotocol_*` fields on target node → `JSON.stringify(data, null, 2)` → `vault.modify()`.
- Native canvas fields (`id`, `x`, `y`, `width`, `height`, `type`, `color`) are never modified.
- On success: also write the primary human-readable field to `text` (e.g., `radiprotocol_questionText` → `text`) so the canvas remains visually readable without reopening the editor.

## Form Field Schema

One form section per node kind. The node type `DropdownComponent` is always rendered first, above the kind-specific section.

### Start node
No kind-specific fields. Display: "Start node has no additional properties."

### Question node
| Field | Label | Obsidian Component | Notes |
|-------|-------|-------------------|-------|
| `radiprotocol_questionText` | "Question text" | `addTextArea` | Required. Synced to native `text` on save. |

### Answer node
| Field | Label | Obsidian Component | Notes |
|-------|-------|-------------------|-------|
| `radiprotocol_answerText` | "Answer text" | `addTextArea` | Required. Synced to native `text` on save. |
| `radiprotocol_displayLabel` | "Display label (optional)" | `addText` | Short label shown in the runner if set. |

### Free-text input node
| Field | Label | Obsidian Component | Notes |
|-------|-------|-------------------|-------|
| `radiprotocol_promptLabel` | "Prompt label" | `addText` | Required. Shown to user above the text field. Synced to `text`. |
| `radiprotocol_prefix` | "Prefix (optional)" | `addText` | Prepended to user's input in accumulated text. |
| `radiprotocol_suffix` | "Suffix (optional)" | `addText` | Appended to user's input in accumulated text. |

### Text-block node
| Field | Label | Obsidian Component | Notes |
|-------|-------|-------------------|-------|
| `radiprotocol_content` | "Content" | `addTextArea` | Required. Auto-appended during session. Synced to `text`. |
| `radiprotocol_snippetId` | "Snippet ID (optional)" | `addText` | Phase 5 field — present in form but labelled "for Phase 5". |

### Loop-start node
| Field | Label | Obsidian Component | Notes |
|-------|-------|-------------------|-------|
| `radiprotocol_loopLabel` | "Loop label" | `addText` | Default: "Loop". Shown as iteration prefix ("Lesion 2"). |
| `radiprotocol_exitLabel` | "Exit label" | `addText` | Default: "Done". Text on the exit button at loop-end. |
| `radiprotocol_maxIterations` | "Max iterations" | `addText` (type=number) | Default: 50. Hard cap. |

### Loop-end node
| Field | Label | Obsidian Component | Notes |
|-------|-------|-------------------|-------|
| `radiprotocol_loopStartId` | "Loop start node ID" | `addText` | Required. Must match an existing loop-start node ID. |

## Accessibility Contract

- All `Setting` instances use `.setName()` for the label — this is Obsidian's built-in accessible label pattern. Do not add `aria-label` manually unless using raw `createEl`.
- All interactive elements must be keyboard-navigable via Tab. The `Setting` API handles this natively.
- The Save button must be the last focusable element in the form for correct tab order.
- Error notices via `new Notice()` are announced by screen readers via Obsidian's notice implementation — no additional `aria-live` region is needed.
- The idle state container must not be focusable (`tabindex` not set).
