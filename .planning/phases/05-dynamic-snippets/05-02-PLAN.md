  ```typescript
  function insertAtCursor(textarea: HTMLTextAreaElement, text: string): void {
    const start = textarea.selectionStart ?? textarea.value.length;
    const end = textarea.selectionEnd ?? start;
    textarea.value = textarea.value.substring(0, start) + text + textarea.value.substring(end);
    textarea.selectionStart = start + text.length;
    textarea.selectionEnd = start + text.length;
    textarea.dispatchEvent(new Event('input'));
  }
  ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input → DOM | Snippet name and template text entered by user, rendered into DOM elements |
| DOM event handlers → SnippetService | Button clicks trigger async vault operations |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-5-05 | Tampering | SnippetManagerView — user-entered name/template | mitigate | All text set via `element.textContent` or `.setText()` / `createEl({ text: ... })` — never `innerHTML`; sanitize() in SnippetService.save() handles control chars before disk write |
| T-5-06 | Denial of Service | [Save] button — rapid clicks | mitigate | Button disabled during save (set `btn.disabled = true` before async call, restore in finally block) |
| T-5-07 | Spoofing | Snippet list order | accept | Snippet list is local to user's own vault; ordering is cosmetic; no security boundary crossed |
| T-5-08 | Tampering | [Delete] two-step confirm | mitigate | Two-step prevents accidental single-click deletes; no additional security boundary beyond usability |
</threat_model>
