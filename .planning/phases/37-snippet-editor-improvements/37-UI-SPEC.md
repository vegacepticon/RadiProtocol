## Color

All colors use Obsidian CSS custom properties, adapting automatically to light/dark themes.

| Role | Obsidian Var | Usage |
|------|-------------|-------|
| Dominant (60%) | `--background-primary` | Tree root background, input backgrounds |
| Secondary (30%) | `--background-secondary` | Expanded chip bg, add-placeholder form bg |
| Accent (10%) | `--interactive-accent` | Primary CTA button (`mod-cta` class) only — reserved for the "Новый" snippet button |
| Hover | `--background-modifier-hover` | Row hover, button hover states |
| Muted text | `--text-muted` | Icons, badges, secondary labels, folder button text color |
| Border | `--background-modifier-border` | Header bottom border, form borders |

Accent reserved for: the existing "Новый" (create snippet) button ONLY. The new "Папка" (create folder) button does NOT use `mod-cta` — it uses plain button styling to maintain visual hierarchy (one primary action, one secondary action).

## Interaction Contract

### Create Folder Flow

1. User clicks "Папка" button in snippet editor header
2. `ConfirmModal` opens with folder name prompt (existing UX)
3. User types folder name, clicks confirm
4. Folder created under `.radiprotocol/snippets/` root
5. Tree redraws with new folder visible (expand-state auto-updated)
6. No Obsidian `Notice` needed on success (folder appears visually)

### Error States

| Error | Trigger | User Sees |
|-------|---------|-----------|
| Empty folder name | User submits blank input | Validation prevents submission (existing behavior in `handleCreateSubfolder`) |
| Folder name with slash | User types `foo/bar` | Validation prevents submission (existing behavior) |
| Folder already exists | User types existing name | `vault.adapter` rejects; existing error handling applies |

### Canvas Ref Sync (SYNC-01) — Already Implemented

No new UI for SYNC-01. The rename-to-canvas-sync pipeline (`commitInlineRename` -> `rewriteCanvasRefs`) was implemented in Phase 34. The existing Notice reads: "Папка переименована. Обновлено канвасов: N". No changes needed.

## CSS Changes

### New rules (append to `src/styles/snippet-manager.css`)

No new CSS classes are needed. The new button reuses the existing `.radi-snippet-tree-new-btn` class, which already provides:
- `display: inline-flex`
- `align-items: center`
- `gap: var(--size-2-1)` (4px between icon and label)

The visual differentiation (secondary vs primary) comes from the absence of `mod-cta` on the folder button. Obsidian's base button styling provides adequate secondary appearance.

If testing reveals the secondary button needs more visual distinction, a single optional rule may be added:

```css
/* Phase 37: folder button secondary styling (only if needed) */
.radi-snippet-tree-header .radi-snippet-tree-new-btn:not(.mod-cta) {
  color: var(--text-muted);
}
```

This is conditional — only add if the default button styling does not provide enough contrast against the `mod-cta` primary button.
