    for (const wallet of DONATE_WALLETS) {
      const desc = wallet.networks ? wallet.networks.join(' · ') : '';
      const { address } = wallet;
      new Setting(containerEl)
        .setName(wallet.name)
        .setDesc(desc)
        .then(setting => {
          setting.descEl.createEl('code', {
            text: address,
            cls: 'rp-donate-address',
          });
        })
        .addExtraButton(btn => btn
          .setIcon('copy')
          .setTooltip(DONATE_TOOLTIP_TEXT)
          .onClick(() => {
            void navigator.clipboard.writeText(address).then(() => {
              new Notice(DONATE_NOTICE_TEXT);
            });
          })
        );
    }

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Settings tab DOM → user clipboard | The copy button transfers the wallet address from the in-memory constant to the OS clipboard. The user is the trust boundary's far side — they paste this into a wallet app to send crypto. |
| Plugin code → Obsidian platform | `new Notice(...)` is the only platform-side write. No filesystem write, no network call, no shell-out. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-71-05 | Spoofing | Address `<code>` elements | mitigate | Full untruncated address shown so users can visually verify before pasting (Row-D-02 anti-phishing decision). The constants test (Plan 01 NTC-D-04) catches typos at build time. The dev-vault UAT (this plan, Task 2) catches DOM-rendering bugs at human-verify time. |
| T-71-06 | Tampering | Clipboard write | mitigate | `navigator.clipboard.writeText(address)` is called with the address pulled directly from the closure-bound `DONATE_WALLETS[i].address`. No string concatenation, no template, no formatting — the address that was loaded into memory is the address that goes onto the clipboard. The Notice "Адрес скопирован" gives the user a confirmation signal that the click registered. |
| T-71-07 | Information disclosure | Settings tab render | accept | The donate section displays only public crypto-wallet addresses (designed to be public) and Russian UI strings. No PII, no secrets. |
| T-71-08 | Denial of service | Clipboard reject (theoretical) | accept | Per Row-D-04 + Phase 31 D-08, `.catch` is intentionally omitted. Obsidian Desktop runs in Electron where `navigator.clipboard.writeText` does not realistically reject. If a future Obsidian version introduces a permission gate that rejects, the worst case is a silent no-op (Notice doesn't fire — user can re-click). The trade-off (one extra try/catch line vs. impossible-scenario handling) was already settled project-wide. |
</threat_model>
