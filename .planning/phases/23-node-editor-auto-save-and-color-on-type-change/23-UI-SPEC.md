All color uses Obsidian CSS custom properties. The plugin adapts to light and dark themes automatically.

| Role | Obsidian var | Usage |

## Component Inventory

New DOM elements introduced in this phase:

### `.rp-editor-saved-indicator`

Replaces `.rp-editor-save-row` (which held the Save button). This element occupies the same layout slot at the bottom of `.rp-editor-panel`.

```
position: in the panel flow, after .rp-editor-form
display: flex
align-items: center
min-height: 32px                      /* same slot height as the old save row */
padding-top: var(--size-4-1)          /* 8px — matches old .rp-editor-save-row */
border-top: 1px solid var(--background-modifier-border)
font-size: var(--font-text-size)      /* 14px */
font-weight: var(--font-normal)       /* 400 */
color: var(--interactive-accent)      /* accent — reserved for this element only */
opacity: 0                            /* hidden at rest */
transition: opacity 150ms ease-out   /* fade-in on show, fade-out on hide */
```

Visibility logic:
1. After each successful `saveNodeEdits()` call: set `opacity: 1` (show).
2. After 2000ms: set `opacity: 0` (fade out).
3. On `loadNode()` (new node loaded): immediately set `opacity: 0`, no transition.

Text content: `"Saved ✓"` — literal string, no icon, no Notice().

### Debounce timer (no new DOM element)

Instance field `private _debounceTimer: ReturnType<typeof setTimeout> | null = null` on `EditorPanelView`. No visual representation. Managed entirely in TypeScript.

### Flush-on-switch (no new DOM element)

Synchronous flush of pending edits before `loadNode()` is called for a new node. No visual change — the indicator shows briefly if flush succeeds, then `loadNode()` clears it immediately.

## Interaction Contract

### Auto-save on field change (debounced)

| Trigger | Any `onChange` on any form field |
| Delay | 800ms debounce (cancel previous timer, start new) |
| On fire | `saveNodeEdits(capturedFilePath, capturedNodeId, capturedEdits)` — values captured at schedule time, not at fire time |
| On success | Show `.rp-editor-saved-indicator` for 2000ms then fade |
| On error | Indicator does NOT show; existing `Notice()` error copy is used (see Copywriting) |

### Immediate save on type-change

| Trigger | `onChange` on the node-type dropdown |
| Delay | None — fires synchronously |
| Edits saved | ALL `{ ...this.pendingEdits }` including color lookup from `NODE_COLOR_MAP` (D-04) |
| Debounce timer | Cancel any pending debounce timer after this immediate save |
| On success | Show `.rp-editor-saved-indicator` for 2000ms then fade |

### Flush-on-switch

| Trigger | User clicks a different canvas node while `_debounceTimer !== null` |
| Action | Cancel timer; call `saveNodeEdits()` synchronously with first node's captured state; then call `loadNode()` for second node |
| If flush fails | `loadNode()` still proceeds — do not block navigation on save error (D-03) |

### Dirty guard removal

`NodeSwitchGuardModal` is deleted. `handleNodeClick()` no longer checks `pendingEdits` size. The same-node early-return stays.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Auto-save indicator (success) | `Saved ✓` |

| Error — invalid JSON | `Canvas file contains invalid JSON — cannot save.` (unchanged) |
| Error — node not found | `Node not found in canvas — it may have been deleted.` (unchanged) |
| Error — live save failed | `Save failed — close the canvas and try again.` (unchanged) |
| Error — write failed | `Could not save — write failed. Check file permissions.` (unchanged) |

Primary CTA: none — the Save button is removed. The phase has no explicit user-initiated save action.

Destructive actions: none in this phase. `NodeSwitchGuardModal` is deleted, which removes the only destructive-confirmation dialog. No replacement confirmation is needed.
