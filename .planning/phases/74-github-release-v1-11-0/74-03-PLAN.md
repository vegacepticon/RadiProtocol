**BRAT install procedure:**
1. Create a new Obsidian vault (or use an existing clean test vault).
2. Install the BRAT plugin from Community Plugins.
3. Open BRAT settings → "Add Beta plugin".
4. Enter `vegacepticon/RadiProtocol` and click "Add Plugin".
5. BRAT fetches the `1.11.0` release assets (`manifest.json`, `main.js`, `styles.css`).
6. Go to Settings → Community Plugins → find "RadiProtocol" → Enable.
7. Verify version shows `1.11.0`.
8. Open the Protocol Runner view (sidebar or command palette).

**What to verify (quick smoke test, not exhaustive UAT):**
- Plugin enables without DevTools errors.
- Settings tab shows "Помочь разработке" section at the top (Phase 71).
- Runner view opens (sidebar or tab).
- If a protocol with a loop node is run, the loop-exit button has a visual accent (Phase 70).
- Inline Runner modal (if triggered) shows no export buttons at complete state (Phase 69).

**If BRAT install fails:**
- Check DevTools Network tab for 404s on release assets.
- Verify the GitHub Release assets are public (not draft).
- Verify `manifest.json` has valid `id`, `name`, `version`, `minAppVersion`.
</context>

  <how-to-verify>
    ### Step 1: Prepare clean vault
    - Create a new Obsidian vault (e.g., `BRAT-TEST-1.11.0`) or use an existing dedicated test vault.
    - **Important:** Do NOT use the dev vault (`TEST-BASE`) where the dev-plugin copy may already exist — that would mask real BRAT install issues.

    ### Step 2: Install BRAT
    - Open Settings → Community Plugins → Turn on Safe Mode OFF.
    - Browse community plugins → search "BRAT" → install → enable.

    ### Step 3: Add RadiProtocol via BRAT
    - Open BRAT settings (Settings → Beta Plugin List, or via command palette).
    - Click "Add Beta plugin".
    - Enter repository: `vegacepticon/RadiProtocol`.
    - Click "Add Plugin".
    - Wait for BRAT to download. Check the BRAT notification for success/failure.

    ### Step 4: Enable RadiProtocol
    - Go to Settings → Community Plugins.
    - Find "RadiProtocol" in the list.
    - Click toggle to enable.
    - **Watch DevTools console** (Ctrl+Shift+I / Cmd+Option+I) during enable for any red errors.

    ### Step 5: Verify version
    - In Community Plugins list, the version shown next to "RadiProtocol" must be `1.11.0`.
    - If version is NOT `1.11.0`, STOP. Report the actual version shown.

    ### Step 6: Quick functional smoke test
    - **Settings check:** Open Settings → RadiProtocol. The very first section should be "Помочь разработке" with nine wallet rows (Phase 71 — DONATE-01).
    - **Runner view check:** Open the Protocol Runner sidebar view (or use command palette → "RadiProtocol: Open Protocol Runner"). The view should open without console errors.
    - **Loop hint check (optional but recommended):** If you have a protocol with a loop node, run it and verify the loop-exit picker button has a subtle visual accent different from body-branch buttons (Phase 70 — LOOP-EXIT-01).
    - **Inline runner check (optional but recommended):** Run a protocol via inline mode and verify that at complete state, there are NO Insert / Copy / Save buttons in the modal body (Phase 69 — INLINE-CLEAN-01).

    ### Step 7: Report
    - If ALL checks pass: type **"approved"**.
    - If ANY check fails: describe the failure and STOP. Do NOT mark BRAT-03 complete until the issue is resolved.
  </how-to-verify>
