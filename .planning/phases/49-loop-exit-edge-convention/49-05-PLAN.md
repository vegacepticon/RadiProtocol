<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 2: UAT — verify Runner exit button caption + body button caption in a real Obsidian vault</name>
  <what-built>
    Phase 49 complete — validator rewritten to label-state convention, runner dispatch + view rewired, shared nodeLabel helper in place, fixtures audited. Build green (Task 1). Tests green (Task 1).
    The behavioural change a radiologist would notice:
    1. A loop whose exit edge reads "готово" (or any non-«выход» word) now renders an exit button with that exact text. Previously the edge had to read literally «выход» or it was silently treated as a body branch.
    2. Body-branch buttons now show the target node's label (e.g. "Size?" from a Question node; first 30 chars of a Text block) instead of the edge's own label or "(no label)".
    3. Saving a canvas with a loop node that has zero labeled outgoing edges, or ≥2 labeled outgoing edges, surfaces a new Russian error in the RunnerView error panel (via GraphValidator).
  </what-built>
  <how-to-verify>
    1. **Prepare a test canvas in TEST-BASE vault** (or any vault with the plugin installed):
       - Create a new canvas.
       - Add nodes: Start → Loop (headerText "Тест цикл") → body text-block "Шаг 1" (back-edge to Loop) → exit text-block "Готово".
       - Connect Loop → "Шаг 1" with NO label (body edge).
       - Connect Loop → "Готово" with label "выполнено" (non-«выход» exit label).
       - Connect "Шаг 1" → Loop (unlabeled back-edge).
       Save the canvas.

    2. **Run the canvas in the Protocol Runner:**
       - Click the "Run" button on the canvas (or use Start-From-Node command).
       - Expected at the loop picker:
         - One button labeled **"выполнено"** (NOT "выход") with the exit-button styling (rp-loop-exit-btn).
         - One button labeled **"Шаг 1"** (the target text-block's content, first 30 chars) with the body-button styling (rp-loop-body-btn).
       - Click the "выполнено" button → session should complete (advance to "Готово" node and finalise).

    3. **Test the validator errors:**
       - **Zero-exit case:** modify the canvas so neither outgoing edge of the Loop node has a label. Save. Re-open / re-run → error panel shows: `Loop-узел "Тест цикл" не имеет выхода. Пометьте ровно одно исходящее ребро — его метка станет подписью кнопки выхода.`
       - **Two-exit case:** label BOTH outgoing edges ("выполнено" and "проверка"). Save. Re-run → error panel shows: `Loop-узел "Тест цикл" имеет несколько помеченных исходящих рёбер: <edge-ids>. Должно быть ровно одно выходное ребро — снимите метки с остальных.` (edge-ids are the internal ids — just verify the message shape matches).
       - **No-body case:** remove the body (unlabeled) edge entirely so only the labeled exit remains. Save. Re-run → error panel shows: `Loop-узел "Тест цикл" не имеет тела — добавьте исходящее ребро без метки.`

    4. **Regression check — existing canvases keep working:**
       - Open any pre-Phase-49 canvas that used `"выход"` as the exit label (e.g. any real protocol canvas in TEST-BASE). Run it. Loop picker must render the exit button as "выход" (trimmed) and the body buttons must render the target-node previews.
       - Confirm no pre-existing canvas is newly broken by the convention change.

    5. **Report back:**
       - PASS: all four scenarios above match the described behaviour.
       - FAIL: describe which step failed, what the button caption / error text actually said, and attach a screenshot.
