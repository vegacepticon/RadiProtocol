<audit_detail>
**unified-loop-missing-exit.canvas — re-audit decision:**

Current content:
```json
"edges": [
  { "id": "e1", "fromNode": "n-start", "toNode": "n-loop" },
  { "id": "e2", "fromNode": "n-loop",  "toNode": "n-q1",   "label": "проверка" },
  { "id": "e3", "fromNode": "n-q1",    "toNode": "n-a1" },
  { "id": "e4", "fromNode": "n-a1",    "toNode": "n-loop" }
]
```

`n-loop` has ONE outgoing edge (`e2`, label=проверка). Under D-05 that's a labeled edge. So `exitEdges.length === 1` — **D-01 (zero labeled) does NOT fire.** Instead `bodyEdges.length === 0` — D-03 (no body) fires. The fixture's intent was "loop without an exit edge" (v1.7 wording) — under Phase 49 that conceptually translates to "loop without an unlabeled body edge" OR "loop without any label" depending on how one reads it.

**Decision:** strip the label from e2 so the fixture unambiguously tests **D-01 (zero labeled)**. This matches the Plan 02 test assertion `expect(errors.some(e => e.includes('не имеет выхода'))).toBe(true)` which is D-01. Alternatively, leaving the label keeps the fixture as a D-03 trigger — but Plan 02 already uses `unified-loop-no-body.canvas` for D-03.

**RESOLUTION: STRIP the `"label": "проверка"` from `e2` in `unified-loop-missing-exit.canvas` so it cleanly exercises D-01.**

(Audit table updated: `unified-loop-missing-exit.canvas` → **STRIP** `"label": "проверка"` on e2.)
</audit_detail>
