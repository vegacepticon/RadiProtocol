<objective>
Unify the RunnerView scroll-pinning architecture: scroll-to-bottom becomes the default behaviour of `renderPreviewZone`, and the entire Phase 47 RUNFIX-02 one-shot flag mechanism is removed (D-09 + D-10). This closes RUNNER-04 SC4 (file-bound Snippet insert pins to bottom) and SC5 (Back pins to bottom) by making them the same code path as every other render.

Purpose: RUNNER-04's roadmap formulation is "matching the existing correct behaviour for Answer insert and directory-bound Snippet insert". Per D-09/D-11 the user explicitly chose the simpler architecture — pin every render to bottom, drop the conditional flag — over a per-call-site fix. This plan delivers exactly that change in `RunnerView` only; per D-12, inline runner mode is excluded (it has no preview textarea — `appendDeltaFromAccumulator` writes to the active note, scroll handling is delegated to Obsidian).

Output:
- `RunnerView.renderPreviewZone` unconditionally sets `textarea.scrollTop = textarea.scrollHeight` inside the existing `requestAnimationFrame` block, AFTER height auto-sizing.
- `RunnerView.pendingTextareaScrollTop` field deleted.
- `RunnerView.capturePendingTextareaScroll` private method deleted.

- Existing tests that spy on `capturePendingTextareaScroll` updated/retired so they no longer reference the removed surface.
