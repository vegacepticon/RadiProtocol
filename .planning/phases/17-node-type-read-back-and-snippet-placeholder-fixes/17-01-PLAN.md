<objective>
Fix BUG-02 and BUG-03: free-text-input and text-block nodes configured via Node Editor
are silently absent from the runner's parsed graph when the canvas is open during save.

Root cause: `openCanvas()` calls `vault.read()` which reads the stale on-disk file. When
`saveLive()` used the Pattern B path (setData + debounced requestSave), the disk file may
not reflect the latest canvas state within the 500ms debounce window. The parser receives
stale JSON without `radiprotocol_nodeType`, silently skips the node, and the runner cannot
find it in the graph.

Fix: Add `getCanvasJSON(filePath: string): string | null` to `CanvasLiveEditor`. In
`openCanvas()`, call `getCanvasJSON()` first — if it returns a non-null string, use that
as the content for parsing. If null (canvas closed), fall back to `vault.read()` as today.

The parser receives the same JSON string type regardless of source — no change to
`CanvasParser.parse()` signature.

Purpose: Ensure runner graph reflects the current Node Editor state at the moment
of opening the protocol, without requiring the user to wait for the debounce or
close the canvas.

Output:
- `getCanvasJSON()` public method on `CanvasLiveEditor`
- `openCanvas()` in `runner-view.ts` updated with live-data fallback
- Passing unit tests (RED then GREEN)
</objective>
