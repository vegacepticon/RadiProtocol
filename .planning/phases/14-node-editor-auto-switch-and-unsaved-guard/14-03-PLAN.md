**Fix A ‚Äî listener accumulation guard (gaps 2 & 3, UAT tests 5 & 6)**

Currently the method begins by unconditionally nulling `watchedCanvasContainer` (line 51-52) and then re-registering a listener every time it is called. Because `active-leaf-change` fires on every modal open/close, each guard interaction adds +2 extra listeners, so the user must dismiss N accumulated modals.

Change the top of `attachCanvasListener()` from:

```typescript
private attachCanvasListener(): void {
  // Bookkeeping null-out only ‚Äî registerDomEvent handles actual cleanup on unload.
  this.canvasPointerdownHandler = null;
  this.watchedCanvasContainer = null;

  const canvasLeaves = this.plugin.app.workspace.getLeavesOfType('canvas');
  const activeLeaf = this.plugin.app.workspace.getMostRecentLeaf();
  const canvasLeaf = canvasLeaves.find(l => l === activeLeaf) ?? canvasLeaves[0];
  if (!canvasLeaf) return;
```

to:

```typescript
private attachCanvasListener(): void {
  const canvasLeaves = this.plugin.app.workspace.getLeavesOfType('canvas');
  const activeLeaf = this.plugin.app.workspace.getMostRecentLeaf();
  const canvasLeaf = canvasLeaves.find(l => l === activeLeaf) ?? canvasLeaves[0];
  if (!canvasLeaf) return;

  // Early-return: already watching this exact container ‚Äî do not re-register.
  // Prevents listener accumulation when active-leaf-change fires on modal open/close.
  if (this.watchedCanvasContainer === canvasLeaf.containerEl) return;

  // Genuine switch to a different canvas leaf ‚Äî reset bookkeeping.
  this.canvasPointerdownHandler = null;
  this.watchedCanvasContainer = null;
```

**Fix B ‚Äî wrong event + unnecessary setTimeout (gap 1, UAT test 1)**

Obsidian canvas commits its internal `selection` Set in response to its own `click` handler, which runs AFTER `pointerdown`. Listening on `pointerdown` with `setTimeout(0)` still reads an empty selection on the first click because the canvas itself hasn't processed the event yet.

Change the handler body and `registerDomEvent` call from:

```typescript
    this.canvasPointerdownHandler = () => {
      setTimeout(() => {
        const selection = canvasView.canvas?.selection;
        if (!selection || selection.size !== 1) return; // ignore multi-select

        const node = Array.from(selection)[0];
        if (!node?.id) return;

        const filePath = canvasView.file?.path;
        if (!filePath) return;

        void this.handleNodeClick(filePath, node.id);
      }, 0);
    };

    this.registerDomEvent(
      this.watchedCanvasContainer,
      'pointerdown',
      this.canvasPointerdownHandler
    );
```

to:

```typescript
    this.canvasPointerdownHandler = () => {
      const selection = canvasView.canvas?.selection;
      if (!selection || selection.size !== 1) return; // ignore multi-select

      const node = Array.from(selection)[0];
      if (!node?.id) return;

      const filePath = canvasView.file?.path;
      if (!filePath) return;

      void this.handleNodeClick(filePath, node.id);
    };

    this.registerDomEvent(
      this.watchedCanvasContainer,
      'click',
      this.canvasPointerdownHandler
    );
```

Also update the field name comment on line 18 from `canvasPointerdownHandler` to reflect it now handles `click` events ‚Äî or leave the field name as-is (it is private and harmless). Prefer leaving the name unchanged to minimize diff noise.

No other changes to the file. Do not touch `handleNodeClick`, `NodeSwitchGuardModal`, `onOpen`, or any other method.
