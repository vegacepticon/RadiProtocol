<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Persisted session file → openCanvas() | Session JSON is read from vault disk — untrusted content if user edits file manually |
| Canvas JSON → parser → runner | Canvas file content → graph model → runner state |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-20-03-01 | Tampering | openCanvas() legacy session guard | mitigate | A session file with a manually-crafted `runnerStatus: "awaiting-snippet-fill"` value is handled safely: the raw-string cast detects it before TypeScript narrowing, sessionService.clear() is called, and a fresh session starts. No user-facing error leaks session content. Implementation uses `(session as unknown as { runnerStatus: string }).runnerStatus` — the cast does not expose any new attack surface. |
| T-20-03-02 | Elevation of Privilege | editor-panel-view.ts free-text-input removal | accept | Removing the 'free-text-input' dropdown option reduces the plugin's editable surface. No privilege escalation vector introduced. |
| T-20-03-03 | Information Disclosure | styles.css | accept | CSS changes are static style rules with no data access. Removing .rp-free-text-input reduces dead code in the stylesheet. |
| T-20-03-04 | Denial of Service | openCanvas() awaiting-snippet-fill guard | accept | The guard calls sessionService.clear() once and falls through. No loop, no I/O retry, no unbounded wait. O(1) path. |
</threat_model>
