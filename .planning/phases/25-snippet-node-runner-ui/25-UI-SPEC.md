## Component Inventory

Phase 25 adds two new components and reuses one existing CSS class:

### 1. File-picker button (runner-view, reused class)

- **Class:** `rp-answer-btn` (no new CSS)
- **Location:** `runner-view.ts` `case 'at-node'` → inner `case 'snippet':` branch
- **Element:** `createEl('button', { cls: 'rp-answer-btn', text: resolvedLabel })`
- **Label resolution:** `node.buttonLabel ?? canvasNodeText ?? "Select file"` (D-09)
- **Behavior on click:** resolve folder → guard empty/unconfigured → open `SnippetFilePickerModal`
- **Primary visual anchor:** this button is the sole interactive element rendered in the `at-node` / snippet state — it is the focal point by exclusivity. No competing elements.
- **New CSS required:** none

### 2. SnippetFilePickerModal (new)

- **Class name:** `SnippetFilePickerModal` (at Claude's discretion — D-08 area)
- **Base class:** `FuzzySuggestModal<TFile>` (Obsidian API)
- **Location:** new file `src/views/snippet-file-picker-modal.ts`
- **Pattern:** mirrors `NodePickerModal` (SuggestModal subclass structure)
- **Display:** filename only — `file.name` (D-01), no path prefix
- **Scope:** all `.md` and `.json` files recursively from configured folder (D-02)
- **Placeholder:** "Search snippet files…"
- **Cancel (Esc):** result promise resolves `null` → runner stays on node (D-03)
- **New CSS required:** none — Obsidian handles all modal chrome styling

### 3. Error/empty notices (reused pattern)

- **API:** `new Notice(message)` (Obsidian native)
- **Triggers:** empty folder (D-04), unconfigured folder (D-05), invalid JSON (D-06)
- **New CSS required:** none
