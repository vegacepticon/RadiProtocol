# Phase 64 — UI Design Contract

> Visual and interaction contract for Node Editor auto-grow fields and Text block quick-create. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — no React/shadcn/new UI library |
| Preset | not applicable |
| Component library | Obsidian DOM API + existing `Setting` patterns only |
| Icon library | Obsidian `setIcon` / Lucide IDs already used by the plugin |
| Font | Obsidian theme font via `var(--font-interface)` / inherited UI font |

**Source:** REQUIREMENTS.md, 64-CONTEXT.md, `editor-panel-view.ts`, `editor-panel.css`. Preserve the current Node Editor visual language and Phase 48 bottom full-width vertical toolbar stack.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps (`var(--size-2-1)`), label-to-field micro gaps |
| sm | 8px | Button row/column gaps, helper text spacing, saved-indicator padding (`var(--size-4-1)`) |
| md | 16px | Panel padding and field block padding (`var(--size-4-2)`) |
| lg | 24px | Toolbar separation from form content (`var(--size-4-3)`) |
| xl | 32px | Not introduced in this phase |
| 2xl | 48px | Not introduced in this phase |
| 3xl | 64px | Not introduced in this phase |

Button padding must use 4px-grid-safe values. If Phase 64 touches toolbar button padding, use `8px 12px` or `8px 16px`; do not introduce non-4px spacing.

## Interaction Contract

### Growable Textareas

- Apply the Phase 48 Question textarea behavior to every authored multi-line Node Editor text field: Question text, Answer text, Text block content, Loop header text, and Snippet branch label.
- On form load, set `height: auto`, then set `height` to `scrollHeight + 'px'` inside `requestAnimationFrame` so populated fields open at full content height.
- On `input`, repeat `height: auto` then `scrollHeight + 'px'` before scheduling auto-save; this must support typing, paste, deletion, and shrinking.
- No inner scrollbar should appear for normal multi-line field content. Use `overflow-y: hidden` for managed growable fields.
- Preserve manual vertical resize affordance only if it does not fight auto-grow; auto-grow must remain authoritative after each input.
- Auto-save keys must remain unchanged: Question `radiprotocol_questionText` + `text`; Answer `radiprotocol_answerText` + `text`; Text block `radiprotocol_content` + `text`; Loop `radiprotocol_headerText` + `text`; Snippet branch `radiprotocol_snippetLabel` with empty string saved as `undefined`.
- Do not overwrite a focused field from Phase 63 live-sync behavior; if Phase 63 is present, auto-grow should run after accepted incoming value updates and skip destructive re-render of the actively edited field.

### Text Block Quick-Create Button

- Add a fifth quick-create button to the existing create-button group, after Loop or before Duplicate. Duplicate remains a separate utility button, not counted as a create kind.
- Button text: `Create text block`; accessible name/title: `Create text block`.
- Icon: use `file-text` or another existing passive text/content Lucide icon; do not introduce a new visual style.
- Click behavior: call the existing `onQuickCreate(...)` path widened to include `'text-block'`; creation must flow through `CanvasNodeFactory.createNode(canvasPath, 'text-block', currentNodeId)`.
- Result: node has `radiprotocol_nodeType = "text-block"`, `NODE_COLOR_MAP['text-block']` color, default placement below the anchor, and immediately renders the Text-block form from live in-memory node data.
- Failure state: preserve existing Notice copy `Open a canvas first to create nodes.`

## Layout and Responsive Contract

- Preserve the Phase 48 bottom toolbar: `.rp-editor-create-toolbar` remains full-width, vertical, below the form content, with top border and `margin-top: var(--size-4-3)`.
- Visual hierarchy: the selected node form remains primary; the bottom quick-create toolbar is secondary, and the new Text block button must match peer create buttons without becoming the focal point.
- All toolbar buttons remain `width: 100%` and `justify-content: center` at normal and narrow sidebar widths.
- Narrow sidebar behavior: the five create buttons plus Duplicate must remain reachable by vertical scrolling inside the Node Editor; no horizontal overflow, clipping, or hidden controls.
- Growable fields must be full width inside the Node Editor form and must not exceed panel width; use `box-sizing: border-box`.
- The form area may scroll as a whole; individual textareas should not introduce inner scrollbars for content growth.

---

## Accessibility Contract

- Every new/converted textarea must have visible label text immediately above or via the existing `Setting` label; helper copy remains visible and muted.
- The Text block button must be keyboard focusable as a native `button`, activated by Enter/Space, and expose `aria-label` + `title` matching visible copy.
- Preserve Obsidian focus rings and theme contrast; do not remove outlines.
- Dynamic textarea growth must not move focus or reset cursor position.
- Screen-reader semantics stay simple: no live region required for height changes; existing `Saved ✓` indicator behavior remains unchanged.
