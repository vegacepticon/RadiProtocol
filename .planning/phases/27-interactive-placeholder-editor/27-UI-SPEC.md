## Design System

| Property | Value |
|----------|-------|
| Tool | none — Obsidian native CSS variable system |
| Preset | not applicable |
| Component library | Obsidian DOM API (`createEl`, `createDiv`) — no external component library |
| Icon library | none — drag handle uses Unicode glyph (⠿ or ⋮⋮, executor discretion) |

Obsidian provides a design token system via CSS custom properties. All spacing, typography, and colour values in this contract reference those tokens. No Tailwind, no shadcn, no React.

## Spacing Scale

Declared values — all multiples of 4, using Obsidian spacing tokens:

| Token | Obsidian CSS var | Resolved value | Usage |
|-------|-----------------|----------------|-------|
| xs | `var(--size-2-1)` | 4px | Icon gaps, inline padding, chip internal gaps |
| sm | `var(--size-4-1)` | 8px | Between chips, header padding, form row gaps |
| md | `var(--size-4-2)` | 16px | Form panel padding, section gaps |
| lg | `var(--size-4-3)` | 24px | Not used in this phase |
| xl | `var(--size-4-4)` | 32px | Not used in this phase |

Exceptions:
- Drag handle touch/click target: minimum 32px height (same as `rp-placeholder-row-header` min-height) — source: CONTEXT.md D-01
- Colour bar strip: exactly 4px wide — source: CONTEXT.md D-01
- Remove button `[×]`: 24×24px clickable target — consistent with `rp-option-row button` in existing styles.css

## Interaction Contract

### Chip Layout Structure

```
[⠿ handle] [color-bar] [label text]  [type badge]  [×]
```

All in a single flex row. The colour bar is achieved via `border-left` on the chip — no separate span DOM node. Handle is a `<span>` at the far left, 24px wide. Remove button is at the far right.

### Click Regions

| Region | Action | Guard condition |
|--------|--------|----------------|
| Drag handle (`rp-placeholder-chip-handle`) | Begin drag (`dragstart`) | Must NOT trigger expand |
| Chip body (label + badge area) | Toggle expand/collapse inline editor | Must NOT trigger when `e.target` is handle or remove button |
| Remove button `[×]` | Remove placeholder, re-render list, check orphans | Must call `e.stopPropagation()` |

Source: CONTEXT.md D-03, D-04, code-context section.

### Drag-and-Drop

Implementation: HTML5 native drag API — no external library (source: CONTEXT.md D-05).

| Event | Element | Behaviour |
|-------|---------|-----------|
| `dragstart` | chip row | `dataTransfer.setData('text/plain', String(index))` |
| `dragover` | sibling chip | `e.preventDefault()` + add `drag-over` class |
| `dragenter` | sibling chip | add `drag-over` class |
| `dragleave` | sibling chip | remove `drag-over` class |
| `drop` | sibling chip | splice `draft.placeholders`, remove `drag-over`, re-render, auto-save |
| `dragend` | chip row | remove `drag-over` from all chips (cleanup guard) |

Drag ghost: browser default. No custom ghost image required.

### Expand / Collapse Inline Editor

- Click chip body → expand inline editor below chip (same form as current: Label, Type dropdown, Options/Unit/Sep)
- Same element `renderExpandedPlaceholder()` / `renderNumberExpanded()` logic reused
- `is-expanded` class on chip row drives background change (same as current `rp-placeholder-row.is-expanded`)

### Auto-Save on Drop

After drop re-renders the list, call `snippetService.save(draft)` immediately.
- Success: `new Notice("Snippet saved.")`
- Failure: `new Notice("Failed to save snippet. Check that the vault is writable and try again.")`
- The [Save snippet] button remains active and still works for other edits (source: CONTEXT.md D-06)
