# Phase 78-01: Pre-commit Hook

## Goal
Install a pre-commit git hook that runs `eslint` on staged `*.ts` files and `npm test`, blocking commits on failure.

## Steps

### Step 1: Create `.githooks/pre-commit`
Shell script that:
1. Gets list of staged `*.ts` files via `git diff --cached --name-only --diff-filter=ACM`
2. If no `*.ts` files staged → exit 0 (skip)
3. Run `npx eslint` on staged `*.ts` files only
4. If lint errors → print error message, exit 1
5. Run `npm test` (full suite)
6. If any test fails → print error message, exit 1
7. Exit 0

Script must be executable (`chmod +x`).

### Step 2: Wire the hook
```bash
git config core.hooksPath .githooks
```

### Step 3: Verify CI-01 (lint error blocks commit)
1. Create a temp branch
2. Introduce a deliberate eslint error in a staged file (e.g., unused variable)
3. Attempt `git commit -m "test: lint error"` → should FAIL with non-zero exit
4. Clean up: revert the change

### Step 4: Verify CI-02 (--no-verify bypass)
1. On the same broken branch with the eslint error staged
2. Run `git commit --no-verify -m "test: bypass"` → should SUCCEED
3. Clean up: reset the commit and revert the change

### Step 5: Verify clean commit works
1. On `main` with clean source
2. Create a harmless staged change
3. `git commit` → should PASS (hook runs lint + test, both green)
4. Amend to remove the test commit or reset

## Constraints
- Do NOT modify `package.json` (no husky/lint-staged)
- Do NOT modify `eslint.config.mjs`
- Hook script must work on Linux/macOS (bash/sh compatible)
- Use `npx eslint` not `npm run lint` to target only staged files
- Full `npm test` runs (not filtered) — vitest is fast enough
