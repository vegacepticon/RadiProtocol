# Plan 84-02 — i18n extraction: views and snippets

## Objective

Replace all remaining hard-coded Russian strings in `src/views/` and `src/snippets/` with `t()` calls.

## Scope

Files to modify:
- `src/views/snippet-editor-modal.ts`
- `src/views/snippet-tree-picker.ts`
- `src/views/snippet-manager/tree-renderer.ts`
- `src/views/snippet-manager-view.ts`
- `src/views/node-picker-modal.ts`
- `src/views/editor-panel/forms/snippet-form.ts`
- `src/views/snippet-chip-editor.ts`
- `src/snippets/snippet-model.ts`
- `src/snippets/snippet-service.ts`

For each file:
1. Identify every user-facing Russian string.
2. Add keys to `src/i18n/locales/en.json` and `src/i18n/locales/ru.json`.
3. Replace the literal with `this.plugin.i18n.t('key')` or pass `plugin` down if needed.
4. Do NOT change test assertion semantics — if a test asserts on a literal Notice text, update the test to assert on the key or use a test-locale helper.

## Constraints

- Keep changes additive: only replace strings, do not refactor unrelated logic.
- `src/views/runner-view.ts` is OUT for this plan — it goes to 84-03 because it has error-message strings that overlap with graph/runner.

## Verification

```bash
npm run build && npm run lint && npm test
```

## Success criteria

- `git grep -nP '[\u0400-\u04FF]' src/views/snippet-editor-modal.ts src/views/snippet-tree-picker.ts src/views/snippet-manager/ src/views/snippet-manager-view.ts src/views/node-picker-modal.ts src/views/editor-panel/forms/snippet-form.ts src/views/snippet-chip-editor.ts src/snippets/snippet-model.ts src/snippets/snippet-service.ts` returns zero matches.
- All tests pass.
