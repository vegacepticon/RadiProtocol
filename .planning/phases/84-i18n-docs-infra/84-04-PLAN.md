# Plan 84-04 — Write README.md

## Objective

Create a repo-root `README.md` suitable for GitHub visitors.

## Scope

Create `README.md` with:
1. Project description and core value (one paragraph).
2. Feature list (bullet points): canvas-based protocol authoring, interactive runner with step-back, dynamic snippets with placeholders, loops for multi-lesion workflows, inline/sidebar/tab runner modes.
3. Installation: BRAT instructions + manual `main.js` + `manifest.json` copy.
4. Quick-start: create a canvas → add question/answer nodes → run protocol.
5. Screenshot placeholders (commented `<!-- screenshot: ... -->`) for future images.
6. License reference (repo LICENSE file).

## Constraints

- No Obsidian community plugin store submission info (still deferred).
- English only (primary).

## Verification

```bash
npm run build && npm run lint && npm test
```
(README does not affect build/test, but gate must still pass.)

## Success criteria

- `README.md` exists at repo root.
- Content covers all 6 required sections.
- No Russian strings in README.
