# Plan 84-03 — i18n extraction: graph, runner, and runner-view error messages

## Objective

Internationalize validation and runtime error messages in the graph, runner, and runner-view layers.

## Scope

Files to modify:
- `src/graph/canvas-parser.ts` — legacy `free-text-input` rejection message
- `src/graph/graph-validator.ts` — loop/snippet validation errors
- `src/runner/protocol-runner.ts` — legacy runtime error
- `src/views/runner-view.ts` — snippet not-found / validation-error strings

For each error message:
1. Add a keyed entry to `src/i18n/locales/en.json` and `ru.json`.
2. Replace the literal with `this.plugin.i18n.t('key', { param: value })`.
3. Update tests that assert on exact message text to assert on the key or use a stubbed i18n helper.

## Constraints

- Do NOT change validation logic — only the error text.
- Interpolation parameters must match between en and ru.

## Verification

```bash
npm run build && npm run lint && npm test
```

## Success criteria

- `git grep -nP '[\u0400-\u04FF]' src/graph/canvas-parser.ts src/graph/graph-validator.ts src/runner/protocol-runner.ts src/views/runner-view.ts` returns zero matches.
- All tests pass.
