  ### Scenario 1 — ROADMAP SC-1: Skip advances without text append

  **Setup:** Open any canvas in TEST-BASE with at least one Question → Answer chain (e.g., a working protocol from Phase 50 UAT). Click through until a Question is active with ≥ 2 answer options visible.

  **Steps:**
  1. **Before clicking Skip**, note the textarea's current accumulator text. If empty, type a manual edit: `PRE-SKIP-MARKER`.
  2. Click the **Skip** icon-button (the one with the `skip-forward` lucide icon below the answer buttons).
  3. Observe the runner advances to the node **after** the first answer (i.e., the first answer's first downstream neighbor).
  4. Check the textarea — it should still say `PRE-SKIP-MARKER` (BUG-01 / D-11 roundtrip: manual edit preserved). **NO answer text appended.**
  5. Click **Step back**.
  6. Observe the runner returns to the original question with its answer buttons visible, and the textarea still says `PRE-SKIP-MARKER` (D-10 roundtrip).

  **Pass criteria:**
  - [ ] Skip button visible at the question (icon-only, no visible text label)
  - [ ] Hovering Skip shows a tooltip ("Skip this question" or similar)
  - [ ] After click: advanced past the question; accumulator text UNCHANGED except for the manual edit marker
  - [ ] After step back: back at the question, accumulator text equals pre-skip state
