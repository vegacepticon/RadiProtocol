# Plan 75-01 — Scaffold shared runner renderer + host interface

## Objective

Create the shared renderer architecture under `src/runner/` without changing runtime behavior.

## Scope

Create:

- `src/runner/runner-host.ts`
- `src/runner/runner-renderer.ts`
- `src/runner/snippet-label.ts`
- `src/runner/runner-text.ts`
- optional scaffold test: `src/__tests__/runner/runner-renderer-scaffold.test.ts`

Do not yet remove rendering logic from `RunnerView` or `InlineRunnerModal` except for tiny type-safe imports if required by scaffold tests.

## Implementation notes

- `RunnerHost` must be interface-only / type-only where possible to avoid circular imports.
- `RunnerRenderer` should be instantiable with stub host + runner.
- `snippet-label.ts` should centralize file-bound vs directory-bound snippet branch labels only if it can be done without behavior change.
- `runner-text.ts` may host shared accumulated-text extraction only if both callers can use it safely.

## Verification

Run:

```bash
npx vitest run runner-renderer-scaffold
npm run build
npm run lint
npm test -- RunnerView inline-runner runner-snippet
```

## Success criteria

- New scaffold modules exist.
- Host shells still behave as before.
- No behavior is moved yet unless fully covered.
- Build/lint/targeted tests pass.
