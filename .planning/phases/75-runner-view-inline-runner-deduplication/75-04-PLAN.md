# Plan 75-04 — Migrate awaiting-snippet-pick / SnippetTreePicker

## Objective

Move duplicated snippet picker rendering and `SnippetTreePicker` mount logic into shared renderer code while preserving host-specific picker behavior.

## Scope

Create/modify:

- `src/runner/render/render-snippet-picker.ts`
- `src/views/runner-view.ts`
- `src/views/inline-runner-modal.ts`

## Behavior to preserve

- Host-specific picker class:
  - RunnerView: `rp-stp-runner-host`
  - InlineRunnerModal: `rp-stp-inline-host`
- Stale-result guard based on captured state/node id.
- Inline modal DOM-attached guard (`document.body.contains(containerEl)`).
- RunnerView deprecated defensive picker path/node fields if still referenced.
- RU/EN validation/not-found copy divergence unless explicitly wrapped by host locale.
- Autosave/delta append side effects after picker selection.

## Verification

Run:

```bash
npx vitest run runner-snippet-picker inline-runner-modal
npm test -- RunnerView inline-runner runner-snippet
npm run build
npm run lint
```

## Success criteria

- `renderSnippetPicker` implementation is declared in exactly one shared renderer module.
- Existing snippet picker tests pass unchanged.
- Host-specific CSS hooks and lifecycle guards are preserved.
