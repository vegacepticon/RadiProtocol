## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user→settings field | User types an arbitrary string into the "Default snippet files folder" text input |
| settings→disk | Serialized settings object written to plugin data.json via plugin.saveSettings() |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-24-01 | Tampering | snippetNodeFolderPath onChange | mitigate | `value.trim()` strips leading/trailing whitespace; stored as plain string with no shell evaluation — used only as a vault-relative path passed to `app.vault.getAbstractFileByPath()` in Phase 25, which is safe |
| T-24-02 | Information Disclosure | settings.json on disk | accept | Settings file is part of the Obsidian vault; all other settings fields carry the same risk; folder paths are not credentials |
| T-24-03 | Elevation of Privilege | path traversal via `../` in folder path | accept (Phase 24) | Phase 24 stores the raw trimmed string only — no file system access occurs in this phase. Phase 25 MUST validate/normalize the path before calling vault APIs (out of scope here) |
| T-24-04 | Denial of Service | empty string causes plugin crash | mitigate | Empty string is the valid "not configured" sentinel (D-01, D-05); no code in Phase 24 acts on the value — consumers (Phase 25) guard with `if (folderPath)` before use |
</threat_model>
