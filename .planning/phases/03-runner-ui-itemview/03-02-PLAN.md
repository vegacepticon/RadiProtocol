     * Called when the user edits the preview textarea directly (RUN-11 / D-04).
     * Manual edits cannot be undone via stepBack() (D-05).

    setAccumulatedText(text: string): void {

      this.undoStack = []; // D-05: clear undo stack on manual edit

      const timestamp = new Date().toISOString().replace(/[:.]/g, '-').slice(0, 19);

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| previewTextarea.value → navigator.clipboard | User-authored text written to system clipboard |
| previewTextarea.value → vault.create() | User-authored text written to vault as a new file |
| outputFolderPath (settings) → vault path | User-configured path used in file system operations |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-02-01 | Tampering | vault.create() with user-supplied path | mitigate | `outputFolderPath` is user-configured in settings (not from canvas data); no directory traversal risk within vault. Obsidian Vault API sanitizes paths. |
| T-03-02-02 | Denial of Service | vault.create() failure | mitigate | Wrapped in try/catch; failure shows Notice and logs console.error — does not crash the view |
| T-03-02-03 | Information Disclosure | navigator.clipboard.writeText | accept | User intentionally copies their own protocol output; no third-party exposure |
| T-03-02-04 | Repudiation | setAccumulatedText clears undo | accept | By design (D-05); user is explicitly editing the text; undo-of-edit is out of scope for v1 |
</threat_model>
