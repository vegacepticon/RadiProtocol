# Phase 3 — UI Design Contract

> Visual and interaction contract for the RunnerView ItemView. Generated by gsd-ui-researcher.
> Consumed by gsd-ui-checker, gsd-planner, gsd-executor, and gsd-ui-auditor.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | Obsidian built-in helpers only (`createEl`, `createDiv`, `Setting`, `Modal`, `SuggestModal`) |
| Icon library | Obsidian built-in via `setIcon()` (Lucide subset bundled with Obsidian) |
| Font | Inherited from Obsidian theme — do not override font-family |
| Style layer | `styles.css` in plugin root — Obsidian CSS custom properties exclusively |

**Constraint (NFR-02):** No React, no Tailwind, no shadcn, no third-party component library.
All DOM construction via `createEl()` / `createDiv()` only. No `innerHTML`, `outerHTML`, or `insertAdjacentHTML`.

---

## Spacing Scale

All spacing values reference Obsidian CSS custom properties where available, falling back to
explicit px values that align to the 4-point grid. No arbitrary values.

| Token | Value | Obsidian var (preferred) | Usage |
|-------|-------|--------------------------|-------|
| xs | 4px | `var(--size-2-1)` | Icon gaps, badge padding, swatch gap |
| sm | 8px | `var(--size-4-1)` | Button vertical padding, inline element gap |
| md | 16px | `var(--size-4-2)` | Zone internal padding, input padding |
| lg | 24px | `var(--size-4-3)` | Zone-to-zone gap, section divider margin |
| xl | 32px | `var(--size-4-4)` | Sidebar outer padding (top/bottom) |
| 2xl | 48px | — | Not used in Phase 3 |
| 3xl | 64px | — | Not used in Phase 3 |

**Exceptions:**
- Answer buttons: min-height 40px (touch-safe; `var(--input-height)` if within range, otherwise explicit).
- Zone divider border: 1px — not a spacing token, a border rule.

---

## Typography

All sizes inherit from Obsidian theme variables. Do not hardcode font-size values in px unless
Obsidian provides no variable.

| Role | Size | Obsidian var | Weight | Line Height | Usage |
|------|------|--------------|--------|-------------|-------|
| Body | 14px | `var(--font-text-size)` | 400 (`var(--font-normal)`) | 1.5 | Preview textarea, answer button labels, legend descriptions |
| Label | 13px | `var(--font-smaller)` | 500 (`var(--font-medium)`) | 1.4 | Question text heading, section sub-labels, legend type names |
| Heading | 16px | `var(--font-ui-medium)` | 600 (`var(--font-semibold)`) | 1.2 | Zone headings ("Question", "Protocol preview"), validation error heading |
| Display | — | — | — | — | Not used in Phase 3 |

**Rules:**
- All UI text in sentence case per NFR-05 (e.g., "Run protocol", "Step back", "Copy to clipboard").
- No bold inside body paragraphs — reserve semibold (600) for headings and labels only.
- The preview `textarea` uses body size (14px / 400 weight / 1.5 lh) with monospace override:
  `font-family: var(--font-monospace)` so medical text (measurements, symbols) is unambiguous.

## Color

All colors resolve to Obsidian CSS custom properties. No hardcoded hex values in plugin CSS.
The 60/30/10 split maps to Obsidian's semantic variable layers.

| Role | Obsidian Variable | Usage |
|------|-------------------|-------|
| Dominant (60%) | `var(--background-primary)` | RunnerView background, textarea background |
| Secondary (30%) | `var(--background-secondary)` | Question zone background, legend section background |
| Accent (10%) | `var(--interactive-accent)` | See reserved-for list below |
| Destructive | `var(--text-error)` / `var(--color-red)` | Validation error text, error state panel border |
| Border | `var(--background-modifier-border)` | Zone divider, textarea border, legend border |
| Interactive hover | `var(--interactive-hover)` | Answer button hover state background |
| Muted text | `var(--text-muted)` | Empty state body copy, placeholder text in textarea and free-text input |

**Accent reserved for:**
1. Primary "Copy to clipboard" button (filled/accent background)
2. Primary "Save to note" button (filled/accent background)
3. Currently selected answer button focus ring (outline, not fill)
4. Active link text in the legend (if any)

Accent is NOT applied to: answer buttons (default state), step-back button, zone backgrounds,
legend swatches, free-text input border.

**Node type color swatches in the legend** use explicit semantic colors (not Obsidian accent).
These are documentation swatches, not interactive elements:

| Node type | Swatch color | Obsidian variable |
|-----------|-------------|-------------------|
| start | Green | `var(--color-green)` |
| question | Blue | `var(--color-blue)` |
| answer | Default | `var(--background-primary)` with border |
| free-text-input | Yellow | `var(--color-yellow)` |
| text-block | Purple | `var(--color-purple)` |
| loop-start / loop-end | Orange | `var(--color-orange)` |

## Layout Contract

### RunnerView — Two-Zone Vertical Split

```
┌─────────────────────────────┐
│  Question zone (flex: 0 0 auto; overflow-y: auto) │
│  - Question text (Label role)                     │
│  - Answer buttons (vertical list, full-width)     │
│  - OR: Free-text input textarea                   │
│  - OR: Text-block auto-advance indicator          │
│  - Step back button (bottom of zone)              │
├─────────────────────────────┤  ← 1px border, var(--background-modifier-border)
│  Preview zone (flex: 1 1 auto; overflow-y: auto)  │
│  - Zone heading: "Protocol preview"               │
│  - Editable textarea (full width, full flex height)│
│  - Output toolbar: [Copy to clipboard] [Save to note] │
├─────────────────────────────┤
│  Legend section (collapsible, collapsed by default)│
│  - 7 rows: swatch + type name + one-line description│
└─────────────────────────────┘
```

**Flex rules (apply to `.rp-runner-view` root container):**
- Root: `display: flex; flex-direction: column; height: 100%; padding: var(--size-4-2)` (16px)
- Question zone: `flex: 0 0 auto; overflow-y: auto; background: var(--background-secondary); border-radius: var(--radius-s); padding: var(--size-4-2);`
- Zone divider: `border-bottom: 1px solid var(--background-modifier-border); margin: var(--size-4-1) 0;` (8px margins)
- Preview zone: `flex: 1 1 auto; display: flex; flex-direction: column; gap: var(--size-2-1);` (4px gap between heading, textarea, toolbar)
- Preview textarea: `flex: 1 1 auto; width: 100%; resize: none;` (fills remaining height)
- Legend section: `flex: 0 0 auto; border-top: 1px solid var(--background-modifier-border); margin-top: var(--size-4-1);`

### Answer Buttons

- One button per answer option, in vertical stack
- `width: 100%; text-align: left; min-height: 40px; padding: var(--size-4-1) var(--size-4-2);` (8px 16px)
- `white-space: normal; word-break: break-word;` — no truncation ever
- Use Obsidian `ButtonComponent` class pattern or `button.mod-cta` for primary, plain button for answers
- Answer buttons: class `mod-quiet` (Obsidian secondary button style) in default state
- No grid layout for answers, no icon-only buttons for answers

### Free-Text Input Node

- Full-width `textarea` with 4 visible rows minimum
- Placeholder: "Type your finding here..." (sentence case)
- Below: one "Continue" button (`mod-cta`)
- `padding: var(--size-4-2);` inside the question zone

### Text-Block Auto-Advance

- No user interaction required — node text displayed in question zone as a read-only paragraph
- Class `rp-text-block-indicator`: italic body text, `var(--text-muted)` color
- Auto-advances after 0 ms (immediate, no animation in Phase 3)

### Validation Error Panel

- Replaces the question zone when validation fails before session start
- Heading: "Protocol cannot be opened" (Heading role, `var(--text-error)`)
- Ordered list of plain-language error messages (one per validation failure) — `<ol>` via `createEl('ol')`
- Below list: one paragraph "Fix the canvas and try again." (`var(--text-muted)`)
- No modal — displayed inline in the question zone

### Idle State (no canvas loaded)

- Question zone shows empty state (see Copywriting Contract below)
- Preview textarea is empty and disabled (`disabled` attribute)
- Output buttons are disabled

### Complete State

- Question zone shows completion message (see Copywriting Contract)
- Preview textarea remains editable
- Output buttons are enabled and accent-colored

## Interaction Contract

### Answer Selection
- Click → `runner.chooseAnswer(answerId)` → re-render question zone → append text to preview textarea
- No double-click, no keyboard shortcut (keyboard nav via native tab + enter is sufficient)
- Button is visually active on `:focus-visible` with `outline: 2px solid var(--interactive-accent)`

### Step Back
- Single "Step back" button, placed at the bottom of the question zone
- Disabled when `runnerState.canStepBack === false`
- Click → `runner.stepBack()` → re-render question zone → revert preview textarea to snapshot text
- Icon: Obsidian `undo-2` (Lucide); label: "Step back"; layout: icon left of text label

### Inline Text Editing (RUN-11)
- Preview textarea is always editable (not `readonly`)
- `input` event → `runner.setAccumulatedText(textarea.value)`
- Editing clears the undo stack (manual edit is not reversible via step-back — this is correct per D-05)
- No visual indicator of "undo stack cleared" — silent behavior

### Copy to Clipboard (UI-05)
- Label: "Copy to clipboard"
- Click → `navigator.clipboard.writeText(accumulatedText)` → momentary button label change to "Copied!" for 1500 ms → reverts to "Copy to clipboard"
- No modal confirmation

### Save to Note (UI-06)
- Label: "Save to note"
- Click → create note at `plugin.settings.outputFolderPath / {timestamp}-protocol.md` → show Obsidian `Notice`: "Protocol saved to {path}"
- If `outputDestination === 'clipboard'` only: button hidden
- If `outputDestination === 'new-note'` only: "Copy to clipboard" hidden
- If `outputDestination === 'both'`: both buttons visible

### Legend Toggle (UI-12)
- Rendered as a `<details>` element with `<summary>Legend</summary>` — native collapse, no JS required
- Collapsed by default (`<details>` without `open` attribute)
- One row per node type: `[swatch 12×12px] [type name, Label role] — [one-line description, Body role, var(--text-muted)]`

### Start from Specific Node Command (RUN-10)
- Exposed via command `start-protocol-from-node` — not from within RunnerView panel
- Opens `SuggestModal` — presents all `question` and `text-block` nodes, displayed by node label
- User picks one; `runner.start(graph, selectedNodeId)` called
- If no canvas is loaded when command fires: `new Notice('Open a canvas protocol first.')` and abort

### Workspace Persistence (UI-07)
- `getState()` returns `{ canvasFilePath: string | null }`
- `setState(state)` restores `canvasFilePath` into RunnerView's idle state (canvas preselected, no session resumed)
- Session resume is Phase 7 only

## Settings Tab Contract (UI-10, UI-11)

All controls use Obsidian `Setting` API. No `createEl('input')` directly for settings.

| Control | API | Label | Description |
|---------|-----|-------|-------------|
| Output destination | `DropdownComponent` | "Output destination" | Options in order: "Clipboard only", "New note only", "Both" |
| Output folder path | `TextComponent` | "Output folder" | Visible only when destination is `new-note` or `both`. Placeholder: "RadiProtocol Output" |
| Max loop iterations | `SliderComponent` | "Max loop iterations" | Range 10–200, step 10. Display current value as text next to slider |

Settings heading: `Setting.setHeading()` with text "RadiProtocol" — no `createEl('h2')` (per ESLint rule `no-manual-html-headings`).

## Accessibility Notes

- Answer buttons must have descriptive text (the answer label IS the accessible name — no `aria-label` override needed)
- Step-back button in disabled state: `button.disabled = true` (native HTML; screen reader announces "dimmed")
- Preview textarea: `aria-label="Protocol preview"` set via `createEl('textarea', { attr: { 'aria-label': 'Protocol preview' } })`
- Validation error list: wrap in `role="alert"` so screen readers announce it on render
- Legend `<details>/<summary>` is natively accessible — no ARIA additions needed
- All interactive elements reachable via Tab key in document order (question zone top → bottom → preview → toolbar → legend)
