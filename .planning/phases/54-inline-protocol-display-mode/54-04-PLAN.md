<task type="checkpoint:human-verify" gate="blocking">
  <what-built>Phase 54 inline protocol display mode: floating modal over active note, command palette launch, answer append-to-note, freeze/resume on note switch, close-on-tab-close, inline snippet/loop/fill-in UI, no regression to sidebar/tab modes.</what-built>
  <how-to-verify>
Open Obsidian with TEST-BASE vault. Ensure a protocol canvas exists in the Protocol folder and a markdown note is open.

**SC-1: Command launch + canvas picker (Tests 1-2)**
1. Open command palette (Ctrl/Cmd+P), type "Run protocol in inline"
2. Verify command appears and is selectable
3. With no active markdown note (switch to a canvas or image), run the command
4. Verify Notice: "Open a markdown note first, then run this command."
5. Switch back to a markdown note, run the command again
6. Verify canvas picker appears listing .canvas files from Protocol folder
7. Select a canvas
8. Verify floating modal appears bottom-right over the active note

**SC-2: Answer append + note as buffer (Tests 3-5)**
9. In the modal, answer a question by clicking an answer button
10. Switch to the target note — verify the answer text was appended to the END of the note
11. Answer another question — verify second answer appended after the first
12. Verify the modal's preview zone shows accumulated text
13. Verify you can edit the note directly while the modal is open (type something in the note)
14. Answer another question — verify new answer appended after your manual edit

**SC-3: Freeze/resume + close-on-tab-close (Tests 6-8)**
15. While the modal is visible, switch to a different note
16. Verify the modal disappears (hidden via is-hidden class)
17. Switch back to the target note
18. Verify the modal reappears at the same node (state preserved)
19. Close the target note tab entirely
20. Verify the modal closes (run ended)

**SC-4: Close button + no confirmation (Test 9)**
21. Mid-run, click the X button on the modal
22. Verify modal closes immediately — NO confirmation dialog
23. Verify partial output remains in the note (not rolled back)

**SC-5: Nested UI + no regression (Tests 10-12)**
24. Run a protocol with a snippet node — verify snippet picker renders INSIDE the modal (not as a centered Obsidian Modal)
25. Run a protocol with a loop node — verify loop picker renders INSIDE the modal
26. Run a protocol with a .json snippet with placeholders — verify fill-in form renders INSIDE the modal
27. Run the same protocol in sidebar mode — verify unchanged behavior
28. Run the same protocol in tab mode — verify unchanged behavior
