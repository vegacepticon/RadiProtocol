## Spacing Scale

All spacing uses Obsidian CSS custom property tokens already present in `src/styles.css`.
The effective pixel values map to the 4-point scale as follows:

| Token | Obsidian Variable | Effective Value | Usage |
|-------|------------------|-----------------|-------|
| xs | `var(--size-2-1)` | 4px | Icon gaps, internal zone gap, answer list gap |
| sm | `var(--size-4-1)` | 8px | Zone gap (replaces `<hr>` divider), button row gap, step-back margin |
| md | `var(--size-4-2)` | 16px | Container padding, textarea padding, question zone padding |
| touch-min | — | 40px | Answer button `min-height` — unchanged from v1.1 |

Source: RESEARCH.md (verified from `src/styles.css` lines 3–9, 57–58, 96–112)

Exceptions:
- Answer buttons retain `min-height: 40px` (touch target floor, not a spacing token)
- Gap between zones in `.rp-runner-view` after layout change: `var(--size-4-1)` (8px), added via `gap` property replacing the `<hr>` divider

## Interaction Contract

### LAYOUT-01: Textarea Auto-Grow

Trigger: `renderPreviewZone(zone, text)` called with any text value (including empty string).

Behavior sequence:
1. `textarea.value = text` — set content
2. `textarea.style.height = 'auto'` — collapse to allow shrink on shorter content
3. `textarea.style.height = textarea.scrollHeight + 'px'` — expand to fit content

CSS precondition: `.rp-preview-textarea` must have `overflow: hidden` and no `flex: 1 1 auto` or `min-height` constraints.

The textarea is read-only during protocol execution — user does not type into it between steps. The auto-grow fires only when the runner advances a step and `renderPreviewZone` is called again with accumulated text.

Test environment note: `scrollHeight` returns 0 in jsdom. Tests must assert `textarea.style.height !== ''` (not a specific pixel value). UAT is the primary verification path for actual pixel sizing.

### LAYOUT-02: Zone Stacking

The three zones stack top-to-bottom with `gap: var(--size-4-1)` (8px) between them via `.rp-runner-view`:
- Preview zone (textarea) — top
- Question zone (prompt + answer buttons + step-back) — middle
- Output toolbar (Copy / Save / Insert) — bottom

No CSS `order` property is used. DOM creation order drives visual order.

### LAYOUT-03: Equal Button Widths

The three buttons in `.rp-output-toolbar` each carry `flex: 1`, so they share toolbar width equally. The toolbar itself is `display: flex; flex-direction: row; gap: var(--size-4-1)`.

### LAYOUT-04: Legend Absence

No legend element exists in the DOM after this phase. Neither tab mode nor sidebar mode renders any node type legend. The runner view contains only: previewZone, questionZone, outputToolbar (plus the canvas selector in headerEl, which is untouched).
