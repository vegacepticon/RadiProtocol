<objective>
Fix race condition in onQuickCreate(): loadNode() reads the canvas file from disk via vault.read(), but canvas.requestSave() (called inside createNode) is async fire-and-forget. loadNode() fires before the file is written, gets stale JSON, and shows "Node not found in canvas".

Purpose: Close the UAT gap from test 3 — newly created nodes must auto-load in the editor immediately.
Output: Patched onQuickCreate with 150ms delay; updated test.
</objective>

In the `onQuickCreate` method (around line 714), add a 150ms delay between `createNode()` and `loadNode()` to give `canvas.requestSave()` time to flush to disk.

Change the current code:

```typescript
    if (result) {
      this.loadNode(canvasPath, result.nodeId);
    }
```

To:

```typescript
    if (result) {
      // Wait for canvas.requestSave() to flush the new node to disk.
      // requestSave() is async fire-and-forget inside createNode(), so
      // loadNode() → renderNodeForm() → vault.read() would get stale JSON
      // without this delay. 150ms is sufficient for Obsidian's save cycle.
      await new Promise(resolve => setTimeout(resolve, 150));
      this.loadNode(canvasPath, result.nodeId);
    }
```

Do NOT modify any other code in editor-panel-view.ts. This is a surgical one-line addition.
