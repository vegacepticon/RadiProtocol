# Phase 33 — UI Design Contract

> Visual and interaction contract for the Snippet Tree view, SnippetEditorModal, folder operations, and vault-watcher driven redraws. Generated by gsd-ui-researcher from CONTEXT.md D-01..D-18. All values below are **prescriptive** — executor must use Obsidian CSS variables as written.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (vanilla TS + Obsidian DOM API) |
| Preset | not applicable |
| Component library | Obsidian `ItemView` / `Modal` / `Menu` / `Notice` / `setIcon` |
| Icon library | Lucide (via Obsidian `setIcon`) |
| Font | Obsidian theme default (`--font-interface`) |

**Rationale:** RadiProtocol is an Obsidian plugin written in vanilla TypeScript (see CLAUDE.md). No React, no shadcn, no third-party UI libraries. All tokens come from Obsidian's CSS custom-property system, which auto-adapts to user theme (light/dark/community themes). This contract locks **which** Obsidian variables to use — executor MUST NOT hardcode colors or px values except where explicitly listed under Exceptions.

---

## Spacing Scale

All spacing uses Obsidian's native scale via CSS variables. Declared tokens (all multiples of 4):

| Token | Obsidian var | Value | Usage in Phase 33 |
|-------|--------------|-------|-------------------|
| xs | `var(--size-2-1)` | 4px | Tree row inner gap (icon↔label), chip gap, indent unit multiplier |
| sm | `var(--size-4-1)` | 8px | Row vertical padding, modal field gap, button gap |
| md | `var(--size-4-2)` | 16px | Row horizontal padding, modal section gap, modal content padding |
| lg | `var(--size-4-3)` | 24px | Modal outer padding, section separators |
| xl | `var(--size-4-4)` | 32px | Empty-state vertical padding |

**Tree indentation:** each nesting level adds **16px** (`var(--size-4-2)`) to `padding-left` of the row. Level 0 = `var(--size-4-2)` (16px), level 1 = 32px, level 2 = 48px, and so on. Chevron icon is included inside that padding band so labels align vertically across siblings.

**Touch target:** tree row minimum height = **28px** (matches Obsidian file-explorer). Hover "+ New" button minimum hit area = 24×24px; icon is 16×16 centered.

**Modal layout:**
- Outer padding: `var(--size-4-3)` (24px) on all sides — Obsidian `Modal` default
- Max-width: **800px** (`modalEl.style.maxWidth = '800px'`) per D-07
- Field-to-field vertical gap: `var(--size-4-2)` (16px)
- Label-to-input gap: `var(--size-4-1)` (8px)
- Button row top margin: `var(--size-4-3)` (24px), border-top `1px solid var(--background-modifier-border)`, padding-top `var(--size-4-2)` (16px)
- Inter-button gap: `var(--size-4-1)` (8px)

**Exceptions:**
- Modal max-width fixed at 800px (not on the 4-grid by design — matches Obsidian's own wide modals).
- Chevron icon size fixed at 12×12px (`setIcon` lucide default for small disclosure triangles; Obsidian convention).

## Typography

All typography inherits Obsidian's `--font-interface` stack. Sizes use Obsidian variables where available.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Tree row label | `var(--font-ui-small)` (~13px) | `var(--font-normal)` (400) | 1.5 | Snippet/folder names in tree rows |
| Modal field label | `var(--font-ui-small)` (~13px) | `var(--font-semibold)` (600) | 1.4 | «Тип», «Папка», «Имя» labels |
| Modal input text | `var(--font-ui-medium)` (~14px) | `var(--font-normal)` (400) | 1.5 | Text input, textarea, dropdown values |
| Modal title | `var(--font-ui-large)` (~16px) | `var(--font-semibold)` (600) | 1.3 | «Новый сниппет» / «Редактирование: {name}» |
| Confirm body | `var(--font-ui-medium)` (~14px) | `var(--font-normal)` (400) | 1.5 | Delete-confirm body copy + file list |
| Inline error | `var(--font-ui-smaller)` (~12px) | `var(--font-normal)` (400) | 1.4 | Collision error under Name field |

**Weight policy:** exactly two weights — `--font-normal` (400) for body/labels/inputs, `--font-semibold` (600) for titles and field labels. No italics, no additional weights.

## Color

RadiProtocol tree UI uses Obsidian's semantic color variables exclusively. No hex literals.

| Role | Obsidian var | Usage |
|------|--------------|-------|
| Dominant (60%) | `var(--background-primary)` | Tree view pane background, modal body background |
| Secondary (30%) | `var(--background-secondary)` | Modal header strip (inherits default), inline error background tint (if used), hover row background via `--background-modifier-hover` |
| Accent (10%) | `var(--interactive-accent)` + `var(--text-on-accent)` | Primary "Сохранить" button in modal, selected tree row left-border (2px), active focus ring |
| Destructive | `var(--text-error)` for label, `var(--background-modifier-error)` for button surface | Delete buttons in confirm modals, collision inline error text |
| Border | `var(--background-modifier-border)` | Modal section separators, chip editor frame, inline error frame |
| Muted text | `var(--text-muted)` | Folder type icons (dimmed), path breadcrumb hint, "…и ещё N элементов" count |
| Normal text | `var(--text-normal)` | Tree labels, modal inputs, button labels |
| Hover surface | `var(--background-modifier-hover)` | Tree row hover, context-menu item hover (handled by Obsidian) |
| Active surface | `var(--background-modifier-active-hover)` | Currently-selected tree row (row being edited in an open modal, if any) |

**Accent reserved for (exhaustive list — nothing else may be accent-colored):**
1. The "Сохранить" / "Save" button in `SnippetEditorModal` (primary CTA, `cta mod-cta` class)
2. The "Создать" / "Create" button in the same modal (mode=create) — same `mod-cta` class
3. The focus ring (`:focus-visible`) on tree rows, text inputs, chevrons, buttons
4. Selected-row left border (2px solid `--interactive-accent`) when a modal is currently editing that row

**Destructive reserved for:**
1. The "Удалить" button in the snippet-delete ConfirmModal (`mod-warning` class)
2. The "Удалить папку" button in the folder-delete ConfirmModal (`mod-warning` class)
3. The inline name-collision error message text under the Name field

All other interactive elements (chevrons, hover "+ New" buttons, context-menu items, chip remove (×), type toggle) use neutral `--text-normal` / `--text-muted` with `--background-modifier-hover` on hover only.

### Interaction states (tree row)

| State | Visual |
|-------|--------|
| Default | `--text-normal` label, transparent background |
| Hover | `background: var(--background-modifier-hover)`, hover "+ Новый" action fades in via opacity 0→1 (150ms) on folder rows only |
| Focus-visible (keyboard) | `outline: 2px solid var(--interactive-accent); outline-offset: -2px` |
| "Currently editing" (modal open for this row) | `border-left: 2px solid var(--interactive-accent); background: var(--background-modifier-active-hover)` |
| Collapsed folder chevron | `chevron-right`, no rotation animation (swap icon via setIcon on toggle) |
| Expanded folder chevron | `chevron-down`, no rotation animation |

**Animation budget (keep minimal):**
- Hover "+ Новый" button opacity: 150ms ease-out
- Modal open/close: Obsidian default (handled by base class)
- Tree expand/collapse: instant re-render (no height animation) — matches Obsidian file-explorer; avoids layout thrash with large folders

### Context menu (Obsidian `Menu` API)

Opened on right-click or long-press. Items order:

**On snippet row:**
1. `Редактировать` → opens `SnippetEditorModal` in edit mode
2. (separator)
3. `Удалить` (destructive icon `trash`) → opens snippet delete confirm

**On folder row:**
1. `Создать сниппет здесь` (icon `plus`) → opens modal with «Папка» pre-filled to this folder
2. `Создать подпапку` (icon `folder-plus`) → inline text prompt (or separate small modal) for folder name, then `service.createFolder`
3. (separator)
4. `Удалить папку` (destructive icon `trash`) → opens folder delete confirm

Styling, hover, keyboard nav are all owned by Obsidian's `Menu` class — do NOT restyle.

### Debounce window (D-13 — discretion)

**Locked: 120ms** for the watcher redraw debounce. Rationale: balances snappy feedback after a single vault event (120ms is imperceptible) against coalescing bursts from multi-file operations (rename-folder fires N events). Implementation: `setTimeout`-based trailing debounce, cleared on each new event, fired on `onClose` if pending.
